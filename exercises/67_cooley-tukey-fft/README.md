### 前置知识：从 DFT 到 FFT

#### 离散傅里叶变换 (DFT)

离散傅里叶变换将时域的 N 个采样点变换为频域的 N 个频谱分量：

```
            N-1
X[k] =      Σ    x[n] · e^(-2πi·k·n/N),   k = 0, 1, ..., N-1
            n=0

其中:
  x[n]  — 时域采样序列 (实数或复数)
  X[k]  — 频域第 k 个分量 (复数)
  N     — 采样点数
  i     — 虚数单位 (i² = -1)
  e^(-2πi·k·n/N) — 旋转因子 (twiddle factor)
```

**直接计算 DFT 的复杂度**：对于每个 k，需要 N 次复数乘法和 N-1 次复数加法。
N 个 k 共需 O(N²) 次复数乘法。当 N=1024 时，约需 100 万次运算；当 N=10⁶ 时
约需 10¹² 次运算——这在实践中是不可接受的。

#### 快速傅里叶变换 (FFT) 的诞生

1965 年，James W. Cooley 和 John W. Tukey 发表了著名的 FFT 算法，
利用"分治" (divide-and-conquer) 策略将 DFT 的计算复杂度从 O(N²) 降至
O(N log₂ N)。

| N    | DFT O(N²)  | FFT O(N log₂ N) | 加速比  |
| ---- | ---------- | --------------- | ------- |
| 8    | 64         | 24              | 2.7×    |
| 256  | 65,536     | 2,048           | 32×     |
| 1024 | 1,048,576  | 10,240          | 102×    |
| 4096 | 16,777,216 | 53,248          | 315×    |
| 10⁶  | 10¹²       | 2×10⁷           | 50,000× |

FFT 被称为"20 世纪最重要的数值算法"，广泛应用于信号处理、图像压缩 (JPEG)、
音频编码 (MP3)、通信系统、雷达、医学成像 (MRI)、地震数据分析等领域。

### Cooley-Tukey FFT 核心思想

#### 分治策略

Cooley-Tukey 算法将 N 点 DFT 分解为两个 N/2 点 DFT：

```
将输入序列 x[n] 按索引奇偶分成两组:

偶数索引: x[0], x[2], x[4], x[6]  → N/2 点 DFT → E[k]
奇数索引: x[1], x[3], x[5], x[7]  → N/2 点 DFT → O[k]

组合公式 (蝶形):
  X[k]       = E[k] + W_N^k · O[k]
  X[k+N/2]   = E[k] - W_N^k · O[k]

其中 W_N^k = e^(-2πi·k/N) 称为"旋转因子" (twiddle factor)。
```

**递归分解**：每个 N/2 点 DFT 又可继续分解为两个 N/4 点 DFT，直到 N=1。
N=8 的 FFT 共有 log₂(8) = 3 级分解。

```
                     N=8 DFT
                   /          \
              N=4 DFT        N=4 DFT
             /      \        /      \
        N=2 DFT  N=2 DFT  N=2 DFT  N=2 DFT
         /    \
    N=1 DFT  N=1 DFT  ...  (共 8 个叶节点)
```

#### 蝶形运算 (Butterfly Operation)

蝶形是 FFT 的基本运算单元，每个蝶形处理两个复数，产生两个输出：

```
蝶形信号流图:

        a (even) ──────┬────── a + W·b  (上支路: 和)
                       │
                       W (旋转因子)
                       │
        b (odd)  ──────┼────── a - W·b  (下支路: 差)
                       │
                       ✕ (交叉)

数学表达:
  upper = a + W · b
  lower = a - W · b

其中:
  a = X[even_idx]  (上输入端)
  b = X[odd_idx]   (下输入端)
  W = W_N^twiddle_idx  (旋转因子)
```

**关键注意事项**：必须先保存 `a` 的原始值，因为计算 `lower` 时需要用到。
如果先更新 `a`，`lower = a_new - W·b` 就会用错误的值。

### 位反转置换 (Bit-Reversal Permutation)

#### 为什么需要位反转？

FFT 的分治过程会自然地打乱输入顺序。如果直接按分治树输入，需要复杂的
索引计算。位反转置换提供了一种优雅的解决方案：

**先按位反转重排输入，然后迭代蝶形运算，无需递归。**

#### N=8 的位反转表

N=8 需要 3 位二进制表示 (2³=8)。位反转将二进制位序颠倒：

```
┌────────┬─────────────┬──────────────┬──────────┐
│ 原索引  │  二进制 (3位) │  位反转二进制  │ 新索引    │
├────────┼─────────────┼──────────────┼──────────┤
│   0    │     000      │     000      │    0     │
│   1    │     001      │     100      │    4     │
│   2    │     010      │     010      │    2     │
│   3    │     011      │     110      │    6     │
│   4    │     100      │     001      │    1     │
│   5    │     101      │     101      │    5     │
│   6    │     110      │     011      │    3     │
│   7    │     111      │     111      │    7     │
└────────┴─────────────┴──────────────┴──────────┘
```

**输入变换**：

```
原始输入:  [1, 2, 3, 4, 4, 3, 2, 1]
              ↓ 位反转重排
位反转后:  [1, 4, 3, 2, 2, 3, 4, 1]
```

#### 高效位反转算法

不使用递归或查表，通过位运算在线计算下一个位反转索引：

```
j = 0;
for (i = 1; i < n; i++) {
    bit = n >> 1;           // 从最高位开始
    while (j & bit) {       // 如果当前位是 1
        j ^= bit;           //   清除当前位
        bit >>= 1;          //   移到下一位
    }
    j ^= bit;               // 翻转当前位
    if (i < j)              // 只交换一次，避免重复
        swap(a[i], a[j]);
}
```

**追踪 N=8**：

```
i=1: j=0→bit=4, j&4=0→j^=4→j=4, 1<4 → swap(a[1],a[4])  (1↔4)
i=2: j=4→bit=4, j&4=4→j^=4→j=0, bit=2→j&2=0→j^=2→j=2, 2<2 → 不交换
i=3: j=2→bit=4, j&4=0→j^=4→j=6, 3<6 → swap(a[3],a[6])  (3↔6)
i=4: j=6→bit=4, j&4=4→j^=4→j=2, bit=2→j&2=2→j^=2→j=0,
     bit=1→j&1=0→j^=1→j=1, 4<1 → 不交换
i=5: j=1→bit=4, j&4=0→j^=4→j=5, 5<5 → 不交换
i=6: j=5→bit=4, j&4=4→j^=4→j=1, bit=2→j&2=0→j^=2→j=3, 6<3 → 不交换
i=7: j=3→bit=4, j&4=0→j^=4→j=7, 7<7 → 不交换
```

#### 各阶段详解

**Stage 1 (group_size=2, half=1, step=4)**：

每组 2 个元素，共 4 组。组内只有 k=0，使用 W8^0=1。

```
组 0: (a[0], a[1]) = (1, 4): T = 1×4 = 4,   upper=1+4=5,  lower=1-4=-3
组 1: (a[2], a[3]) = (3, 2): T = 1×2 = 2,   upper=3+2=5,  lower=3-2=1
组 2: (a[4], a[5]) = (2, 3): T = 1×3 = 3,   upper=2+3=5,  lower=2-3=-1
组 3: (a[6], a[7]) = (4, 1): T = 1×1 = 1,   upper=4+1=5,  lower=4-1=3

结果: [5, -3, 5, 1, 5, -1, 5, 3]
```

**Stage 2 (group_size=4, half=2, step=2)**：

每组 4 个元素，共 2 组。k=0 用 W8^0=1，k=1 用 W8^2=-i。

```

  k=0: (a[0], a[2]) = (5, 5):     T = 1×5 = 5,       upper=5+5=10,   lower=5-5=0
```

整组 8 个元素。k=0 用 W8^0=1, k=1 用 W8^1=√2/2-i√2/2, k=2 用 W8^2=-i,
k=3 用 W8^3=-√2/2-i√2/2。


### 旋转因子 (Twiddle Factor) 详解

旋转因子 W_N^k 是 FFT 的核心，定义为单位圆上的复数：

```
W_N^k = e^(-2πi·k/N) = cos(2πk/N) - i·sin(2πk/N)

在复平面上:
                  Im
                  ↑
          W8^2=-i │
      W8^3        │        W8^1
   (-√2/2-√2/2i)  │     (√2/2-√2/2i)
         \        │        /
          \  135° │  45°  /
           \      │      /
            \     │     /
             \    │    /
              \   │   /
               \  │  /
                \ │ /
                 \│/
    ──────────────●──────────────→ Re
                 /│\        W8^0=1
                / │ \
               /  │  \
              /   │   \
             /    │    \
            /     │     \
           /  225°│ 315° \
          /       │       \
         /        │        \
   W8^5=-W8^1     │       W8^7=-W8^3
                  │
              W8^6=-W8^2=i
```

**N=8 旋转因子表**：

```
┌──────┬─────────────────────┬──────────────────────┐
│ W8^k │     直角坐标         │     极坐标           │
├──────┼─────────────────────┼──────────────────────┤
```

### 频谱分析：时域到频域

#### 本题输入信号的频谱特征

输入序列 `{1, 2, 3, 4, 4, 3, 2, 1}` 是一个实数信号，且**关于 n=3.5 镜像对称**（满足 x[n] = x[7−n]）：

```
    x[n]
    4 |        ●───●
    3 |      ●       ●
    2 |    ●           ●
    1 |  ●               ●
      +─────────────────────→ n
      0  1  2  3  4  5  6  7
```

> **对称性辨析（重要）**：这是**关于半整数点 n=3.5** 的镜像对称，而**不是循环偶对称**。
> 循环偶对称要求 x[n] = x[(N−n) mod N]（即 x[1] 应等于 x[7]），但这里 x[1]=2 ≠ x[7]=1。
> 只有循环偶对称的实信号频谱才是纯实数；本题信号相当于把一个零相位偶信号平移了
> 3.5 个样点，因此其频谱带**线性相位**，是复数（见下表 X[1]、X[3] 的虚部）而非纯实数。
> 频谱幅度仍满足实输入的 Hermitian 共轭对称 X[N−k] = conj(X[k])。

**FFT 输出的频域结果**：

```
┌──────┬─────────────────────────┬──────────────────────────────┐
│ 索引  │  频域值 X[k]             │  物理意义                     │
├──────┼─────────────────────────┼──────────────────────────────┤
│ X[0] │  20.000000 + 0.000000i  │  DC 分量 (直流)               │
│ X[1] │  -5.828427 - 2.414214i  │  基频 (1 周期/8 采样)         │
│ X[2] │   0.000000 + 0.000000i  │  2 次谐波 → 零!              │
│ X[3] │  -0.171573 - 0.414214i  │  3 次谐波                     │
│ X[4] │   0.000000 + 0.000000i  │  奈奎斯特频率 → 零!           │
│ X[5] │  -0.171573 + 0.414214i  │  5 次谐波 (X[3] 的共轭)      │
│ X[6] │   0.000000 + 0.000000i  │  6 次谐波 (X[2] 的共轭)      │
│ X[7] │  -5.828427 + 2.414214i  │  7 次谐波 (X[1] 的共轭)      │
└──────┴─────────────────────────┴──────────────────────────────┘
```

#### 幅度谱

```
|X[k]| = sqrt(real² + imag²)

|X[0]| = 20.000          ← DC 分量最大 (信号均值 × N = 2.5 × 8 = 20)
|X[1]| = 6.308           ← 基频分量
|X[2]| = 0.000           ← 零!
|X[3]| = 0.448           ← 3 次谐波很小
|X[4]| = 0.000           ← 零!
|X[5]| = 0.448           ← 与 |X[3]| 相同 (共轭对称)
|X[6]| = 0.000           ← 零!
|X[7]| = 6.308           ← 与 |X[1]| 相同 (共轭对称)
```

#### 实输入信号的共轭对称性

对于实数输入序列，DFT 满足 **Hermitian 对称性**（共轭对称）：

```
X[N-k] = conj(X[k])  即  X[N-k].real = X[k].real
                         X[N-k].imag = -X[k].imag

验证:
  X[7] = -5.828 + 2.414i
  conj(X[1]) = conj(-5.828 - 2.414i) = -5.828 + 2.414i = X[7] ✓

  X[6] = 0 + 0i
  conj(X[2]) = conj(0 + 0i) = 0 + 0i = X[6] ✓
```

**推论**：对于实数输入，只需计算 X[0] 到 X[N/2]（奈奎斯特频率），
后半部分由对称性自动得到。这就是为什么实际 FFT 库（如 FFTW）
对于实数输入可以节省约一半的计算量。

#### DC 分量验证

X[0] 是所有时域采样之和（旋转因子 W_N^0 = 1）：

```
X[0] = Σ x[n] = 1+2+3+4+4+3+2+1 = 20 ✓
```

这与 FFT 输出的 X[0] = 20.000000 一致。

### 复数运算基础

FFT 涉及大量复数运算，需要正确实现：

#### 复数加法

```
(a + bi) + (c + di) = (a+c) + (b+d)i
```

#### 复数减法

```
(a + bi) - (c + di) = (a-c) + (b-d)i
```

#### 复数乘法

```
(a + bi) × (c + di) = (ac - bd) + (ad + bc)i

推导:
  (a + bi)(c + di)
  = ac + adi + bci + bdi²
  = ac + (ad+bc)i + bd(-1)
  = (ac - bd) + (ad + bc)i
```

**在蝶形运算中的应用**：

```c
/* T = W * odd */
T.real = W.real * odd.real - W.imag * odd.imag;
T.imag = W.real * odd.imag + W.imag * odd.real;

/* upper = even + T */
a[even_idx].real = even.real + T.real;
a[even_idx].imag = even.imag + T.imag;

/* lower = even - T */
a[odd_idx].real = even.real - T.real;
a[odd_idx].imag = even.imag - T.imag;
```

### 实现指南

#### TODO 1: bit_reverse()

使用高效位运算算法对复数数组进行位反转置换。

```
算法步骤:
1. 初始化 j = 0
2. 遍历 i 从 1 到 n-1:
   a. bit = n / 2
   b. while (j 的第 log2(bit)+1 位为 1):
        j 清除该位, bit 右移
   c. j 翻转当前位
   d. if (i < j): 交换 a[i] 和 a[j]
```

**关键点**：

- `i < j` 的判断确保了每对元素只交换一次
- 对于 N=8，位反转结果为：[0,4,2,6,1,5,3,7]
- 不要硬编码 8 个位置的映射——使用通用算法

#### TODO 2: butterfly()

实现单级蝶形运算。

```
参数计算:
  group_size = 1 << stage       // 2, 4, 8
  half       = group_size >> 1  // 1, 2, 4
  step       = N / group_size   // 4, 2, 1

嵌套循环:
  for (g = 0; g < N; g += group_size)        // 外层: 遍历每组
    for (k = 0; k < half; k++)               // 内层: 组内蝶形对
      even_idx    = g + k
      odd_idx     = g + k + half
      twiddle_idx = k * step

      // 保存 even 的原始值 (关键!)
      even_saved = a[even_idx]

      // 复数乘法: T = W[twiddle_idx] × a[odd_idx]
      T.real = W[twiddle_idx].real * a[odd_idx].real
             - W[twiddle_idx].imag * a[odd_idx].imag
      T.imag = W[twiddle_idx].real * a[odd_idx].imag
             + W[twiddle_idx].imag * a[odd_idx].real

      // 蝶形更新
      a[even_idx] = even_saved + T
      a[odd_idx]  = even_saved - T
```

**关键点**：

- 必须先保存 `a[even_idx]` 到局部变量
- 复数乘法的实部和虚部公式不要搞混
- `twiddle_idx` 由 `k * step` 决定，不是 `k`

### Makefile 填写指南

Makefile 有 5 个变量和 1 条编译命令需要填写：

```
变量:
  CC      = gcc          ← C 编译器
  CFLAGS  = -Wall -std=c99 ← 编译选项 (开启警告, C99 标准)
  LDFLAGS = -lm          ← 链接数学库 (防御性; 本题运行期不调用 cos/sin, 可省)
  TARGET  = fft          ← 生成的可执行文件名
  SRC     = fft.c        ← 源文件名

编译命令:
  $(CC) $(CFLAGS) -o $(TARGET) $(SRC) $(LDFLAGS)

（脚手架 Makefile 只有 all / clean 目标；判分/自测请用 clings run 67 或 clings watch，
 查看期望输出用 clings tests 67。）
```

**注意（通用链接规则）**：一般而言 `-lm` 应放在源文件**之后**，因为链接器按顺序解析符号；
若程序确实调用了 `cos`/`sin` 等数学函数而 `-lm` 放在源文件之前，就可能报未定义符号。
不过本题旋转因子是硬编码常量表、运行期并不调用任何 libm 函数，因此 `-lm` 只是防御性链接，
放在哪里、甚至去掉都不影响本题构建。

### 常见错误与调试

#### 错误 1：蝶形运算未保存原始 even 值

```c
/* 错误！*/
a[even_idx].real = a[even_idx].real + T.real;  // a[even_idx] 已被修改
a[odd_idx].real  = a[even_idx].real - T.real;  // 使用了错误的值！

/* 正确！*/
Complex even_saved = a[even_idx];  // 先保存
a[even_idx].real = even_saved.real + T.real;
a[odd_idx].real  = even_saved.real - T.real;
```

#### 错误 2：复数乘法公式记错

```c
/* 错误：把减法写成了加法 */
T.real = w.real * odd.real + w.imag * odd.imag;  // 应该是 ac - bd

/* 正确 */
T.real = w.real * odd.real - w.imag * odd.imag;  // ac - bd
T.imag = w.real * odd.imag + w.imag * odd.real;  // ad + bc
```

#### 错误 3：twiddle_idx 计算错误

```c
/* 错误：直接使用 k 作为旋转因子索引 */
twiddle_idx = k;  // 只在 stage 3 (step=1) 时正确

/* 正确 */
twiddle_idx = k * step;  // step = N / group_size
```

#### 错误 4：位反转交换条件遗漏

```c
/* 错误：无条件交换，导致交换两次恢复原状 */
swap(a[i], a[j]);  // 可能交换两次！

/* 正确：只在 i < j 时交换 */
if (i < j) {
    Complex tmp = a[i];
    a[i] = a[j];
    a[j] = tmp;
}
```

#### 错误 5：忘记链接数学库

```
$ gcc -Wall -std=c99 -o fft fft.c
/usr/bin/ld: undefined reference to 'cos'
/usr/bin/ld: undefined reference to 'sin'

解决: 加上 -lm
$ gcc -Wall -std=c99 -o fft fft.c -lm
```

#### 错误 6：打印格式不匹配

```c
/* 错误：没有使用 %+ 标志，正数前不会显示 + 号 */
printf("%.6f%.6fi", c.real, c.imag);  // 输出：5.0000000.000000i

/* 错误：宽度不够 */
printf("%.6f%+.6fi", c.real, c.imag);  // 输出：5.000000+0.000000i (未对齐)

/* 正确 */
printf("%10.6f%+10.6fi", c.real, c.imag);  // 输出：5.000000 +0.000000i
```

#### 错误 7：stage 循环边界

```c
/* 错误：stage 从 0 开始 */
for (stage = 0; stage < 3; stage++)  // group_size 计算会出错

/* 正确：stage 从 1 开始 */
for (stage = 1; stage <= 3; stage++)  // group_size = 2, 4, 8
```

### FFT 的历史与影响

#### 时间线

```
1805  Gauss      手稿中描述了类似 FFT 的算法 (未发表)
1942  Danielson   提出分治 DFT 的思想
1965  Cooley & Tukey  发表标志性论文, FFT 正式诞生
      "An Algorithm for the Machine Calculation of Complex Fourier Series"
      发表于 Mathematics of Computation, Vol. 19, pp. 297-301
1970s 数字信号处理 (DSP) 芯片出现, FFT 硬件化
1980s JPEG/MPEG 标准采用 DCT (基于 FFT 的变体)
1990s FFTW (Fastest Fourier Transform in the West) 开源库发布
2000+ GPU FFT (CUFFT), 分布式 FFT 在大数据中应用
```

#### 为什么 FFT 如此重要？

1. **信号处理**：将时域信号转换为频域，便于滤波、压缩、分析
2. **图像压缩**：JPEG 使用离散余弦变换 (DCT)，是 FFT 的近亲
3. **音频处理**：MP3、AAC 使用改进的离散余弦变换 (MDCT)
4. **通信系统**：OFDM (4G/5G/WiFi) 使用 FFT/IFFT 实现多载波调制
5. **科学计算**：求解偏微分方程、分子动力学模拟
6. **大数据**：多项式乘法 (FFT 加速)、大整数乘法 (Schönhage-Strassen)

### 扩展思考

#### 1. 位反转序 (Bit-Reversed Order) vs 正常序

Cooley-Tukey 算法有两种实现方式：

| 方式           | 输入顺序 | 输出顺序 | 常用场景     |
| -------------- | -------- | -------- | ------------ |
| DIT (时域抽取) | 位反转   | 正常序   | 本题采用     |
| DIF (频域抽取) | 正常序   | 位反转   | 某些硬件实现 |

DIT (Decimation-In-Time)：先位反转输入，蝶形后得到正常序输出。
DIF (Decimation-In-Frequency)：正常序输入，蝶形后得到位反转输出。

#### 2. 原地计算 (In-Place Computation)

本题的 FFT 是原地算法：蝶形运算直接在原数组上更新，不需要额外存储。
对于 N=8，只需要 8 个 Complex 元素的数组。对于 N=1024，也只需 1024 个。
这使得 FFT 在内存受限的嵌入式系统中也能高效运行。

#### 3. 扩展到更大的 N

本题 N=8，但如果扩展到 N=16、N=1024 或更大，算法结构完全相同：

```
stages = log2(N)
旋转因子表需要 N/2 个条目
每次蝶形遍历 N/2 对元素
总复杂度: O(N log N)
```

#### 4. 逆变换 (IFFT)

逆 FFT (IFFT) 只需将旋转因子取共轭并将结果除以 N：

```
W_N^k → conj(W_N^k) = e^(+2πi·k/N)
最后: X[k] = X[k] / N
```

IFFT 在 OFDM 通信和信号重建中至关重要。

### 知识点总结

| 知识点             | 说明                                         |
| ------------------ | -------------------------------------------- |
| DFT 定义           | 时域到频域的离散变换                         |
| Cooley-Tukey 分治  | 将 N 点 DFT 分解为两个 N/2 点 DFT            |
| 蝶形运算           | FFT 的基本计算单元，两个复数 → 两个复数      |
| 旋转因子 (Twiddle) | W_N^k = e^(-2πi·k/N)，单位圆上的复数         |
| 位反转置换         | 重排输入使蝶形对相邻，支持原地迭代计算       |
| 复数运算           | 加法、减法、乘法 (ac-bd) + (ad+bc)i          |
| 频谱对称性         | 实输入 → Hermitian 对称：X[N-k] = conj(X[k]) |
| DC 分量            | X[0] = 所有采样之和                          |
| 奈奎斯特频率       | X[N/2] 对应最高可分辨频率                    |
| 时间复杂度         | O(N log N) vs DFT 的 O(N²)                   |

### 讨论问题

1. 为什么 X[2] 和 X[4] 为零？这与输入信号的什么特性有关？
2. 只有当实数输入满足**循环偶对称**（x[n] = x[(N−n) mod N]）时，FFT 结果才全是实数。
   本题输入 `{1,2,3,4,4,3,2,1}` 关于 n=3.5 **镜像对称**（x[n]=x[7−n]），但**不满足循环偶对称**
   （x[1]=2 ≠ x[7]=1），因此频谱带线性相位、是复数（有虚部）。请解释"循环偶对称"与
   "关于半整数点的镜像对称"的区别，以及为何后者会引入线性相位。（提示：位反转只是 FFT
   内部按 N 决定的数据重排，不改变 DFT 的数学结果，与频谱是否有虚部无关。）
3. 位反转置换中，为什么需要 `if (i < j)` 条件？如果去掉会发生什么？
4. 如果 N 不是 2 的幂（例如 N=6），Cooley-Tukey 算法还能直接使用吗？
   有哪些处理方法？
5. 在蝶形运算中，旋转因子索引 `twiddle_idx = k * step` 是如何推导出来的？
6. FFT 结果中的共轭对称性有什么实际用途？可以节省多少计算量？
