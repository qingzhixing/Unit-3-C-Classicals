## Lesson 52 — B+ 树索引

### 课程任务

实现 B+ 树索引（ORDER=3, MAX_KEY=2），完成以下 7 个核心函数：

1. **`create_node(is_leaf)`** — 分配并初始化节点（叶子/内部节点）
2. **`insert_into_leaf(leaf, key, value)`** — 在有序叶子中插入键值对
3. **`split_leaf_child(parent, idx, child)`** — 分裂满叶子节点（3 key → 2+1）
4. **`insert_rec(node, key, value)`** — 递归插入，处理溢出与分裂
5. **`insert(root, key, value)`** — 插入入口：空树建叶 / 递归插入 / 根溢出建新根
6. **`search(root, key)`** — 从根走到叶子，线性查找 key
7. **`print_tree(root, depth)`** — 递归缩进打印树结构

`insert_into_inner`（内部节点插入）、`free_tree`、`main` 已提供。

### 前置知识：什么是 B+ 树？

B+ 树是一种多路平衡搜索树，是数据库索引（如 MySQL InnoDB）的核心数据结构。

**B+ 树 vs 普通 B 树**：

```
B 树（所有节点存数据）:          B+树（仅叶子存数据）:
       [10,v0 | 20,v1]                 [10 | 20]       ← 内部节点只存 key（路由）
      /      |       \                /      |      \
  [5,v2] [15,v3] [25,v4]        [5,v2|10,v0] [15,v3|20,v1] [25,v4]  ← 叶子存 key+value
                                 ↑                              ↑
                                 └──────── next ────────────────┘  ← 链表连接

B+树的关键特性:
  1. 内部节点只存 key（索引项），不存 value — 一个内部节点可容纳更多路由键
  2. 所有 key+value 都在叶子节点 — 查找总是走到叶子
  3. 叶子节点通过 next 指针形成有序链表 — 高效范围查询
  4. 所有叶子在同一层 — 完美平衡，O(log n) 查找
```

**为什么数据库用 B+ 树而不是二叉搜索树？**

```
二叉搜索树（磁盘不友好）:              B+树（磁盘友好）:
        50                               [20 | 50 | 80]    ← 一个节点多个 key
       /  \                              /    |    |    \
      20  80                          [5,10] [30,40] [60,70] [90,99]
     / \  / \
    ... ... ...                        一个节点 = 一个磁盘页（4KB~16KB）
                                       一次 I/O 读取多个 key
每个节点一次磁盘 I/O，               树高大幅降低，I/O 次数少
100 万数据 → 树高 ~20               100 万数据 → 树高 ~3（阶数=100）
```

### 本课 B+ 树规格

```
ORDER  = 3           ← 阶数（每个节点最多 ORDER 个子节点）
MAX_KEY = ORDER - 1 = 2   ← 每个节点最多 2 个 key
MIN_KEY = ORDER / 2 = 1   ← 每个节点最少 1 个 key（根除外）

内部节点结构:
  is_leaf = 0
  keys[0], keys[1]           ← 最多 2 个路由键
  children[0], children[1], children[2]  ← 最多 3 个子节点
  例如: [15 | 20] → children[0]管理 <15, children[1]管理 [15,20), children[2]管理 ≥20

叶子节点结构:
  is_leaf = 1
  keys[0], keys[1]           ← 最多 2 个键
  values[0], values[1]       ← 对应的值
  next → 下一个叶子           ← 链表指针
  例如: [5,v2 | 10,v0] → next → [15,v3 | 20,v1] → next → [25,v4] → NULL
```

**重要**: 数组多 1 位（`keys[MAX_KEY+1]`, `values[MAX_KEY+1]`, `children[ORDER+1]`），用于在分裂前临时存放溢出元素。

### 插入操作逐步跟踪

以本题的插入序列为例：insert(10,v0), insert(20,v1), insert(5,v2), insert(15,v3), insert(25,v4)

#### 步骤 1: insert(10, v0)

```
树为空 → 创建叶子节点:
  [leaf] keys: 10 | values: 100
```

#### 步骤 2: insert(20, v1)

```
从根（叶子）插入 20，pos=1（20 > 10）:
  [leaf] keys: 10 20 | values: 100 200

num_keys=2，未超过 MAX_KEY=2，无需分裂。
```

#### 步骤 3: insert(5, v2) — 第一次分裂！

```
从根（叶子）插入 5，pos=0（5 < 10）:
  [leaf] keys: 5 10 20 | values: 50 100 200

num_keys=3 > MAX_KEY=2 → 溢出！需要分裂。

分裂过程（split_leaf_child）:
┌──────────────────────────────────────────────────┐
│ 分裂前: child = [5,v2 | 10,v0 | 20,v1]           │
│                                                    │
│ 1. 创建 new_leaf                                  │
│ 2. 右半复制: new_leaf->keys[0] = child->keys[2]   │
│             = 20                                  │
│             new_leaf->values[0] = child->values[2]│
│             = 200                                 │
│ 3. child->num_keys = 2 (保留 [5,10])              │
│    new_leaf->num_keys = 1 (保留 [20])             │
│ 4. 链表: child->next = new_leaf                   │
│ 5. 上移 key = new_leaf->keys[0] = 20              │
└──────────────────────────────────────────────────┘

根节点溢出 → 创建新根（内部节点）:
        [inner] keys: 20
        /                \
  [leaf] keys: 5 10   [leaf] keys: 20
  values: 50 100      values: 200

内部节点路由规则（keys[pos] <= key 时 pos++）:
  - key=5:  5 <= 20 → pos=0, 走 children[0] ✓
  - key=10: 10 <= 20 → pos=0, 走 children[0] ✓
  - key=20: 20 <= 20 → pos=1, 走 children[1] ✓
  - key=25: 25 > 20  → pos=1, 走 children[1] ✓
```

#### 步骤 4: insert(15, v3) — 第二次分裂！

```
从根（内部节点）开始:
  key=15, 根 keys=[20], 15 <= 20 → pos=0, 走 children[0]

到达叶子 [5,v2 | 10,v0]，插入 15:
  pos=2（15 > 10）
  [leaf] keys: 5 10 15 | values: 50 100 150

num_keys=3 > MAX_KEY=2 → 溢出！分裂:

┌──────────────────────────────────────────────────┐
│ 分裂前: child = [5,v2 | 10,v0 | 15,v3]           │
│                                                    │
│ 右半复制: new_leaf = [15,v3]                      │
│ child 保留: [5,v2 | 10,v0]                        │
│ 上移 key = 15                                     │
│                                                    │
│ 父节点插入 key=15 和 new_leaf:                    │
│   父节点 [20] 在 idx=0 插入 15                    │
│   → [inner] keys: 15 20                           │
│      children[0]=[5,10], children[1]=[15],        │
│      children[2]=[20]                             │
└──────────────────────────────────────────────────┘

结果:
        [inner] keys: 15 20
        /        |        \
  [leaf]      [leaf]     [leaf]
  keys: 5 10  keys: 15   keys: 20
  vals:50 100 vals:150   vals:200
```

#### 步骤 5: insert(25, v4) — 无分裂

```
从根开始: key=25, 根 keys=[15,20], 25 > 20 → pos=2, 走 children[2]

到达叶子 [20,v1]，插入 25:
  pos=1（25 > 20）
  [leaf] keys: 20 25 | values: 200 250

num_keys=2，未超过 MAX_KEY=2，无需分裂。

最终 B+树:
        [inner] keys: 15 20
        /        |        \
  [leaf]      [leaf]     [leaf]
  keys: 5 10  keys: 15   keys: 20 25
  vals:50 100 vals:150   vals:200 250
  next→      next→      next→NULL
```

### 查找操作跟踪

#### search(15)

```
从根开始:
  cur = [inner] keys: 15 20
  key=15, pos: 15 <= 15 → pos=1
  cur = children[1] = [leaf] keys: 15

到达叶子:
  遍历: keys[0]=15 == 15 → 返回 values[0]=150 ✓
```

#### search(30)

```
从根开始:
  cur = [inner] keys: 15 20
  key=30, pos: 30 > 15 → pos=1; 30 > 20 → pos=2
  cur = children[2] = [leaf] keys: 20 25

到达叶子:
  遍历: keys[0]=20 ≠ 30, keys[1]=25 ≠ 30
  → 返回 -1（未找到）✓
```

### B+ 树的分裂规则总结

```
节点满（num_keys > MAX_KEY）时的分裂策略:

叶子分裂（3 个 key → 2+1）:
  ┌──────────┬──────────┬──────────┐
  │ keys[0]  │ keys[1]  │ keys[2]  │
  │ values[0]│ values[1]│ values[2]│
  └──────────┴──────────┴──────────┘
         ↓
  ┌───────────────────┐   ┌──────────┐
  │ keys[0]   keys[1] │   │ keys[2]  │  ← 右半第一个 key 上移
  │ values[0] values[1]│   │ values[2]│
  └───────────────────┘   └──────────┘
  左半 (num_keys=2)       右半 (num_keys=1)
                              ↑
                      up_key = keys[0]

内部节点分裂（3 个 key + 4 个孩子 → 1+1+1+2）:
  ┌──────┬──────┬──────┐
  │  k0  │  k1  │  k2  │    ← k1 上移到父节点
  ├──────┼──────┼──────┤
  │  c0  │  c1  │  c2  │  c3
  └──────┴──────┴──────┘
         ↓
  ┌──────┐          ┌──────┐
  │  k0  │   k1↑    │  k2  │
  ├──────┤          ├──────┤
  │  c0  │  c1      │  c2  │  c3
  └──────┘          └──────┘
  左半               右半
  (num_keys=1)      (num_keys=1)
```

### 叶子链表与范围查询

B+ 树的关键优势——叶子节点形成有序链表：

```
[leaf] → next → [leaf] → next → [leaf] → next → NULL
keys: 5 10      keys: 15      keys: 20 25

范围查询 [12, 25]:
  1. 从根走到起始叶子: 12 ≤ 15 → 走 children[0]?
     不对——需要找到第一个 ≥ 12 的叶子。
     实际做法: 从根走到最左叶子 [5,10]，沿链表遍历。
  2. [5,10]: 5<12✗, 10<12✗ → 跳过
  3. next→[15]: 15≥12 ✓ → 输出 15→150
  4. next→[20,25]: 20≥12 ✓ → 输出 20→200, 25→250
  5. next→NULL → 结束

链表让范围查询只需 O(log n + k)，k 为结果数量。
无链表则需要回溯遍历，复杂度高得多。
```

### 常见错误

| 错误                                       | 后果                                           | 正确做法                                               |
| ------------------------------------------ | ---------------------------------------------- | ------------------------------------------------------ |
| 忘记数组 +1 导致越界                       | keys[2] 写入 values[0] 位置（union），数据错乱 | 使用 `MAX_KEY+1` 和 `ORDER+1` 的数组                   |
| 分裂时未维护 next 链表                     | 范围查询断裂                                   | `new_leaf->next = child->next; child->next = new_leaf` |
| 内部路由用 `<` 而非 `<=`                   | key 等于分隔键时走错子树                       | 使用 `keys[pos] <= key` 判断                           |
| 根溢出时未创建新根                         | 树高度不增长，数据丢失                         | insert() 中检测 overflow 并建新根                      |
| insert_into_leaf 只移动 keys 不移动 values | keys 和 values 错位                            | for 循环中同时移动 keys[i] 和 values[i]                |
| 叶子分裂后 num_keys 设置错误               | 后续插入越界或数据丢失                         | 3 个 key 时：左半 2 个，右半 1 个                      |
| search 中内部路由规则与 insert 不一致      | 查找与插入走不同路径，找不到已插入的 key       | 两个函数使用相同的 `keys[pos] <= key` 规则             |

### 重要知识点

- **B+ 树 = 多路平衡树 + 叶子链表**: 结合了平衡树的查找效率和有序链表的范围查询能力
- **内部节点只存 key**: 提高分支因子，降低树高，减少磁盘 I/O
- **所有 key+value 在叶子**: 查找总是走到叶子层，查找代价稳定
- **分裂是高度增长的唯一途径**: 从叶子向上传播，到根时树高 +1
- **叶子链表是 B+ 树的灵魂**: 支持 O(log n + k) 的范围查询，无需回溯

### 课堂讨论

1. **为什么 B+ 树用 ORDER=3 这么小？** — 教学目的。实际数据库（MySQL InnoDB）的 B+ 树阶数通常是几十到几百，一个节点正好填满一个磁盘页（16KB）。

2. **B+ 树 vs 跳表（Skip List）？** — 都是有序索引结构，跳表实现简单但空间开销大（多层指针），B+ 树磁盘友好。Redis 用跳表，MySQL 用 B+ 树，各有适用场景。

3. **B+ 树如何处理重复 key？** — 通常允许重复，插入时 `keys[pos] <= key` 的路由规则让重复 key 落在最右叶子。叶子中可以存多个相同 key 的不同 value。

4. **删除操作去哪了？** — B+ 树的删除比插入更复杂（可能合并节点），但原理类似——从叶子删除后，若节点太空（< MIN_KEY）则向兄弟借或合并。本题聚焦插入与查找，删除留给扩展学习。

### 数据结构的内存布局

```
BPTreeNode 的内存布局（ORDER=3, MAX_KEY=2）:

┌──────────┬──────────┬─────────────────────┬─────────────────────┬──────────┐
│ is_leaf  │ num_keys │ keys[0] keys[1]     │ values[0] values[1] │  next    │
│  (int)   │  (int)   │ keys[2] (溢出位)    │ values[2] (溢出位)  │ (ptr)    │
│          │          │                     │  或 children[0..3]  │          │
└──────────┴──────────┴─────────────────────┴─────────────────────┴──────────┘

union 让 values 和 children 共享同一块内存:
  - 叶子节点: 使用 values[0..2]，children 不使用
  - 内部节点: 使用 children[0..3]，values 不使用
  - 溢出位 keys[2] 在分裂前临时存放第 3 个 key
```
