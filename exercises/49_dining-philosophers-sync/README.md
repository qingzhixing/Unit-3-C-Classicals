## Lesson 49 — 哲学家就餐问题：状态机模拟 + 非对称防死锁【标杆题】

### 课程任务

模拟五位哲学家围坐圆桌就餐：每人需要左右两根筷子才能进餐。在**单线程状态机**中轮询每位哲学家的状态变化，使用非对称拿筷子策略杜绝死锁。每人吃满 **3 次**后模拟结束，输出每轮的哲学家状态。

实现 `can_eat()`、`pickup()`、`putdown()` 和 `main()` 中的模拟循环。

### 前置知识：死锁的四个必要条件

死锁发生的四个条件（**缺一不可**）：

```
    (1) 互斥 Mutex        — 资源同一时刻只能被一人持有
    (2) 持有并等待 H & W   — 持有已有资源 + 等待新资源
    (3) 不可剥夺 No Preempt — 资源只能由持有者主动释放
    (4) 循环等待 Circular   — 存在一条资源等待环
```

**哲学家就餐问题的死锁场景**：若五人同时左手优先——

```
             P4
       筷4       筷0
    P3                P0
       筷3       筷1
    P2     筷2     P1

    每人左手优先执行步骤:
    P0: 拿起筷0，等待筷1  ─┐
    P1: 拿起筷1，等待筷2   │
    P2: 拿起筷2，等待筷3   ├─ 循环等待环！
    P3: 拿起筷3，等待筷4   │
    P4: 拿起筷4，等待筷0  ─┘
    → 死锁！所有人都在等，无人能释放
```

### 非对称策略：打破循环等待

让 **一位哲学家右手优先**（本项目选 P4），其余左手优先：

```
             P4 ← 先拿右手(筷0)再拿左手(筷4)
       筷4       筷0
    P3 左手优先       P0 左手优先
       筷3       筷1
    P2 左手优先  筷2  P1 左手优先

    效杲: P4 和 P0 同时饿时，都先抢筷0 → 只有一人能拿到
         等待环被打破 → 死锁永远不发生 ✓
```

```
can_eat(id):
  left  = id
  right = (id + 1) % 5

  if id != 4:
      return chop[left] == FREE && chop[right] == FREE   // 左→右
  else:
      return chop[right] == FREE && chop[left] == FREE   // 右→左
```

### 状态机模型

每位哲学家在三种状态间轮回：

```
        计时到        拿到筷子
 THINKING ────→ HUNGRY ────→ EATING
    ↑                            │
    └────── 吃完(放筷子) ────────┘
```

模拟采用**三阶段轮询**（单线程状态机，免 pthread）：

```
FOR 每一轮:
  ① 阶段1: EATING 的哲学家，计时到 → 放筷子 → THINKING
  ② 阶段2: HUNGRY 的哲学家，若能拿筷 → 拿筷 → EATING
  ③ 阶段3: THINKING 的哲学家，计时到 → HUNGRY
```

### 数据结构总览

```
        chopstick[0]  chopstick[1]  chopstick[2]  chopstick[3]  chopstick[4]
             ↑            ↑            ↑            ↑            ↑
         ┌───┴────────┐ ┌─┴──────────┐ ┌─┴──────────┐ ┌─┴──────────┐ ┌─┴────────┐
         │   P0       │ │   P1       │ │   P2       │ │   P3       │ │   P4     │
         │ state[0]   │ │ state[1]   │ │ state[2]   │ │ state[3]   │ │ state[4] │
         │ timer[0]   │ │ timer[1]   │ │ timer[2]   │ │ timer[3]   │ │ timer[4] │
         │ eat_cnt[0] │ │ eat_cnt[1] │ │ eat_cnt[2] │ │ eat_cnt[3] │ │ eat_cnt[4]│
         └────────────┘ └────────────┘ └────────────┘ └────────────┘ └──────────┘

   chopstick[i] = -1        → 空闲
   chopstick[i] = philosopher_id → 被该哲学家持有
   timer[i] = 倒计时（轮）
```

### 第一步详解：初始状态 (Round 0)

```
timer 值: P0=3 P1=2 P2=3 P3=5 P4=2  (srand(42) 计算)
状态:     P0:T P1:T P2:T P3:T P4:T

Round 0: 无人可吃（都 THINKING），仅状态展示
```

### 第二步详解：Round 1 → 有人变饿

```
P1 timer 2→1, P4 timer 2→1  (其他未到)
状态: P0:T P1:T P2:T P3:T P4:T
→ 仍无人变饿
```

### 第三步详解：Round 2 → 第一次进餐尝试

```
P1 timer 1→0 → HUNGRY;  P4 timer 1→0 → HUNGRY
P1 (左手优先): chop[1]==-1 ✓, chop[2]==-1 ✓ → can_eat → 等待阶段2
P4 (右手优先): chop[0]==-1 ✓, chop[4]==-1 ✓ → can_eat → 等待阶段2
状态: P0:T P1:H P2:T P3:T P4:H
```

### 第四步详解：Round 3 → 第一次真正进餐

```
阶段1: 无 EATING
阶段2: P1:H → 拿筷1,筷2 → EATING (timer=1)
       P4:H → 拿筷0,筷4 → EATING (timer=1)
       P2 timer 2→0 → HUNGRY 但 check 筷2(被P1拿着) ✗ → 等待
       P3 timer 4→0 → HUNGRY 但 check 筷3(-1)✓, 筷4(被P4拿着) ✗ → 等待
阶段3: P0 timer 2→1
状态: P0:T P1:E P2:H P3:H P4:E
```

### 死锁 vs 非对称：量化对比

| 维度             | 全左手优先（死锁版）  | 非对称 P4 右手优先（本实现） |
| ---------------- | --------------------- | ---------------------------- |
| 死锁概率         | 一定条件下 100%       | **0%** — 循环等待被打破      |
| 最多并发进餐人数 | 2（不相邻才能同时吃） | 2（理论上限未变）            |
| 公平性           | 死锁后全部饿死        | 轮询公平，每人 3 次          |
| 复杂度           | 同样的代码量          | 多一个 if/else 分支          |
| 思路             | 无防御                | **破坏循环等待条件**         |

### 常见错误与陷阱

| 错误                                          | 为什么错                         | 正确做法                               |
| --------------------------------------------- | -------------------------------- | -------------------------------------- |
| P4 也左手优先                                 | 触发循环等待→死锁                | P4 必须右手优先                        |
| can_eat 只检查一根筷子                        | 两根缺一不可                     | 必须两根筷子都空闲才返回 1             |
| putdown 只放一根筷子                          | 吃完应释放两根                   | 左右筷子都设为 -1                      |
| 阶段顺序错误（先 HUNGRY→EAT 再 EATING→THINK） | 刚吃完饭的筷子还没放就被别人拿走 | 先放筷子（阶段 1），再抢筷子（阶段 2） |
| 忘写 `rand() % 4 + 2`                         | 思考时间无变化                   | 用 rand() 产生 2~5 的随机持续          |
| 不用 srand(42)                                | 输出不可复现 → 测试对不上        | 固定随机种子                           |

### 重要知识点

- **死锁四条件**: 互斥、持有等待、不可剥夺、循环等待——四个齐备才死锁
- **循环等待**: 五人左手同时拿筷子形成等待环，核心死锁原因
- **非对称策略**: 让一个进程改变资源请求顺序，打破循环等待环
- **状态机模拟**: 单线程下用三阶段轮询模拟并发，无需 pthread
- **不可剥夺**: 哲学家手中的筷子不能被抢走，只能自己放下→需要预防而非事后恢复
- **srand(42)** : 固定种子让伪随机序列可复现，是自动化测试的常用技巧
- **三阶段顺序**: 先用完再抢（避免死锁分析混乱）

### 课堂讨论

1. **还有其他防死锁方案吗？** 至少三种：(a) 资源分级编号——始终先拿编号小的筷子；(b) 限制最多 4 人上桌——给第 5 人留出筷子；(c) 引入仲裁者——由服务员分配筷子。本课选方案 (a) 的变体。

2. **为什么是 P4 右手优先而不是 P0？** 选谁都可以，关键是打破对称性。选 P4 只是因为它是编号最大的一位。

3. **单线程模拟和真实多线程有何不同？** 真实多线程中，状态检查→拿筷子不是原子操作，可能在 check 和 grab 之间被调度走，需要互斥锁或原子操作保护。本模拟是简化教学版。

4. **如果不用随机种子会怎样？** 用 `srand(time(NULL))` 导致每次输出不同，自动化判分困难。这也是工程中"可复现测试"的通用原则。

5. **进餐时间设为 1 轮合理吗？** 这是教学简化。实际想让模拟快速收敛。增大 EAT_TIME 会使总轮数线性增长但不影响正确性。

6. **怎么验证没有死锁？** 本算法中 P4 右手优先，使得 P0 和 P4 不可能同时持有筷 0 并等待另一根→循环等待环必然断开→死锁不可能发生。
