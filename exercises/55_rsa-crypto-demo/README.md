## 2. 前置知识

### 2.1 模运算基础

模运算（Modular Arithmetic）是 RSA 的数学基础。核心概念：

- **同余**: a ≡ b (mod n) 表示 a 和 b 除以 n 的余数相同，即 n | (a - b)
- **模加法**: (a + b) mod n = ((a mod n) + (b mod n)) mod n
- **模乘法**: (a * b) mod n = ((a mod n) * (b mod n)) mod n
- **模幂**: a^e mod n = (((a mod n)^e) mod n)

### 2.2 欧拉函数 φ(n)

欧拉函数 φ(n) 定义为 [1, n] 中与 n 互质的整数个数。

对于两个不同素数 p 和 q：

```
φ(p*q) = (p-1) * (q-1)
```

本题中：φ(3233) = φ(61 × 53) = 60 × 52 = 3120

RSA 依赖欧拉定理：若 gcd(a, n) = 1，则

```
a^φ(n) ≡ 1 (mod n)
```

由此可以推导出加解密正确性的核心：

```
若 e*d ≡ 1 (mod φ(n))，即 e*d = k*φ(n) + 1，则
m^(e*d) ≡ m^(k*φ(n) + 1) ≡ (m^φ(n))^k * m ≡ 1^k * m ≡ m (mod n)
```

### 2.3 最大公约数与互质

```
gcd(a, b) = 1  ⇔  a 和 b 互质
```

公钥指数 e 必须满足 gcd(e, φ(n)) = 1，否则无法找到模逆元 d。

### 2.4 素性测试的重要性

RSA 的安全性依赖于两个大素数的乘积难以分解。但在密钥生成阶段，
我们必须确认所选的 p 和 q 确实是素数——这就是素性测试的作用。

对于大数（数百位），试除法不可行。Miller-Rabin 是实际应用中最常用
的素性测试方法，它是基于费马小定理的概率算法，但通过选择适当的见证人
集合可以对特定范围给出确定性结果。

### 2.5 费马小定理与 Miller-Rabin 的直觉

费马小定理指出：若 p 是素数且 gcd(a, p) = 1，则

```
a^(p-1) ≡ 1 (mod p)
```

Miller-Rabin 利用这个定理的"逆否"形式：如果存在某个 a 使得
a^(n-1) ≢ 1 (mod n)，则 n 一定是合数。但费马测试有"伪素数"
(Carmichael 数) 的漏洞——Miller-Rabin 通过额外检测平方根来弥补。

## 3. 算法深度剖析

### 3.1 欧几里得算法 (gcd) — TODO 1

计算两个非负整数的最大公约数。基于以下原理：

```
gcd(a, b) = gcd(b, a mod b)  当 b ≠ 0
gcd(a, 0) = a
```

**手动追踪 gcd(17, 3120)**：

```
步骤 │  a   │  b   │ a % b │ 说明
─────┼──────┼──────┼───────┼────────────
  1  │ 17   │ 3120 │  17   │ 交换: gcd(3120, 17)
  2  │ 3120 │  17  │   9   │ 3120 = 183*17 + 9
  3  │  17  │   9  │   8   │ 17 = 1*9 + 8
  4  │   9  │   8  │   1   │ 9 = 1*8 + 1
  5  │   8  │   1  │   0   │ 8 = 8*1 + 0
  6  │   1  │   0  │   —   │ b==0, 返回 a=1 ✓
```

### 3.2 扩展欧几里得算法 (ext_gcd) — TODO 2

除了求 gcd，还找到 Bézout 系数 x, y 使得：

```
a*x + b*y = gcd(a, b)
```

当 gcd(a, b) = 1 时，x 是 a 模 b 的乘法逆元（即 a⁻¹ mod b）。

**递归推导**：

```
设 ext_gcd(b, a%b) 返回 x1, y1，满足：
  b*x1 + (a%b)*y1 = gcd(b, a%b) = gcd(a, b)

记 a = q*b + r，其中 q = a/b, r = a%b
代入: b*x1 + r*y1 = gcd(a, b)

由 r = a - q*b:
  b*x1 + (a - q*b)*y1 = gcd(a, b)
  a*y1 + b*(x1 - q*y1) = gcd(a, b)

对比 a*x + b*y = gcd(a, b):
  x = y1
  y = x1 - q*y1 = x1 - (a/b)*y1
```

**递归调用栈追踪 ext_gcd(17, 3120)**：

```
调用 ext_gcd(17, 3120)
  → ext_gcd(3120, 17)
    → ext_gcd(17, 9)
      → ext_gcd(9, 8)
        → ext_gcd(8, 1)
          → ext_gcd(1, 0) = 1, x1=1, y1=0  ← 基准 (b==0)
        回溯: x = 0, y = 1 - (8/1)*0 = 1         → (x=0, y=1)
      回溯: x = 1, y = 0 - (9/8)*1 = -1           → (x=1, y=-1)
    回溯: x = -1, y = 1 - (17/9)*(-1) = 2         → (x=-1, y=2)
  回溯: x = 2, y = -1 - (3120/17)*2 = -367        → (x=2, y=-367)
回溯: x = -367, y = 2 - (17/3120)*(-367) = 2      → (x=-367, y=2)

最终: 17*(-367) + 3120*2 = -6239 + 6240 = 1 ✓
```

**从 x 计算私钥 d**：

```
d ≡ x (mod φ)
d = (-367) mod 3120 = 3120 - 367 = 2753
验证: 17 * 2753 = 46801,  46801 mod 3120 = 1 ✓
```

### 3.3 快速模幂运算 (fast_pow) — TODO 3

计算 base^exp mod mod 的高效算法。直接连乘 exp 次是 O(exp)，不可行。
二分求幂（Binary Exponentiation）利用指数的二进制展开：

```
base^exp = base^(b₀·2⁰ + b₁·2¹ + ... + bₖ·2ᵏ)
         = base^(b₀·2⁰) * base^(b₁·2¹) * ... * base^(bₖ·2ᵏ)
```

其中 bᵢ 是 exp 的二进制位。只需要计算 base^(2ⁱ)（通过反复平方），
再乘上 bᵢ=1 的那些项。

```
┌───────────────────────────────────────┐
│          fast_pow(base, exp, mod)     │
├───────────────────────────────────────┤
│                                       │
│  result = 1                           │
│  base = base % mod                    │
│                                       │
│  while exp > 0:                       │
│    ┌─────────────────────┐            │
│    │ if exp & 1:         │ ── 最低位? │
│    │   result = result   │            │
│    │          * base     │            │
│    │          % mod      │            │
│    └─────────────────────┘            │
│    exp = exp >> 1        ── 右移一位   │
│    base = base * base    ── 平方      │
│           % mod                       │
│                                       │
│  return result                        │
└───────────────────────────────────────┘
```

**追踪 fast_pow(42, 17, 3233)**：

```
17 的二进制: 10001 (bit0=1, bit4=1)

  exp=17 (10001b)  bit=1  result = 1*42 % 3233 = 42      base = 42² % 3233 = 1764
  exp=8  (01000b)  bit=0  result = 42                    base = 1764² % 3233 = 2998
  exp=4  (00100b)  bit=0  result = 42                    base = 2998² % 3233 = 1652
  exp=2  (00010b)  bit=0  result = 42                    base = 1652² % 3233 = 2444
  exp=1  (00001b)  bit=1  result = 42*2444 % 3233 = 2557  base = 2444² % 3233 = ...

  结果: 2557
```

### 3.4 Miller-Rabin 素性测试 (is_prime) — TODO 4

Miller-Rabin 是基于以下数学事实的概率素性测试：

**核心洞察**：若 n 是奇素数，且 n-1 = d·2^s（d 为奇数），则对任意
1 < a < n-1，以下两个条件之一必成立：

1. a^d ≡ 1 (mod n)
2. 存在 r ∈ [0, s-1] 使得 a^(d·2^r) ≡ -1 (mod n)

如果某个 a 使两个条件都不成立，则 n 一定是合数。这样的 a 称为
"合数见证人"（witness）。

```
┌────────────────────────────────────────────────────────────┐
│            Miller-Rabin 算法流程                             │
├────────────────────────────────────────────────────────────┤
│                                                            │
│  输入: 待测试整数 num                                       │
│                                                            │
│  ┌─ if num < 2:  return 0 (非素数)                         │
│  ├─ if num == 2 or 3:  return 1 (素数)                     │
│  ├─ if num is even:  return 0 (偶数合数)                    │
│  │                                                         │
│  ├─ 分解 num-1 = d * 2^s:                                  │
│  │   d = num - 1; s = 0;                                   │
│  │   while (d & 1) == 0: d >>= 1; s++;                     │
│  │                                                         │
│  ├─ 对每个见证人 a ∈ {2, 3, 5, 7, 11}:                     │
│  │   ┌─ if a >= num: continue (跳过)                       │
│  │   ├─ x = a^d mod num  (调用 fast_pow)                   │
│  │   ├─ if x == 1 or x == num-1: continue (此见证人通过)    │
│  │   ├─ for r = 0 to s-2:                                  │
│  │   │   x = x^2 mod num                                   │
│  │   │   if x == num-1: break (通过)                       │
│  │   ├─ if x != num-1: return 0 (合数!)                    │
│  │                                                         │
│  └─ return 1 (所有见证人通过 → 素数)                        │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

**见证人选择原理**：

| 见证人集合               | 确定性范围              | 来源            |
| ------------------------ | ----------------------- | --------------- |
| {2}                      | n < 2,047               | 基础费马测试    |
| {2, 3}                   | n < 1,373,653           | —               |
| {2, 3, 5}                | n < 25,326,001          | —               |
| {2, 3, 5, 7}             | n < 3,215,031,751       | —               |
| {2, 3, 5, 7, 11}         | n < 2,152,302,898,747   | Jaeschke (1993) |
| {2, 3, 5, 7, 11, 13}     | n < 3,474,749,660,383   | —               |
| {2, 3, 5, 7, 11, 13, 17} | n < 341,550,071,728,321 | —               |

本题使用 {2, 3, 5, 7, 11}，对所有 n < 2^64（≈ 1.8×10^19）给出确定性结果，
远覆盖 uint64_t 的范围。

**示例：用 Miller-Rabin 验证 61 是素数**

```
num = 61
num-1 = 60 = 15 * 2^2  →  d = 15, s = 2

见证人 a=2:
  x = 2^15 mod 61 = 32768 mod 61 = 11
  x ≠ 1 且 x ≠ 60, 继续循环 r=0:
  x = 11^2 mod 61 = 121 mod 61 = 60 = num-1 ✓ (通过)

见证人 a=3:
  x = 3^15 mod 61 = ...  (类似过程，通过)

... 所有见证人通过 → 61 是素数 ✓
```

## 5. 边界情况分析

| 场景                 | 输入/条件              | 期望行为                  | 说明                                 |
| -------------------- | ---------------------- | ------------------------- | ------------------------------------ |
| m = 0                | 明文为 0               | c = 0, m' = 0             | 0^e mod n = 0, 0^d mod n = 0         |
| m = 1                | 明文为 1               | c = 1, m' = 1             | 1^anything = 1                       |
| m = n-1              | 明文为 3232            | c = (n-1)^e mod n         | 仍正确解密（欧拉定理保证）           |
| gcd(e, φ) ≠ 1        | e 与 φ 不互质          | gen_keys 返回错误         | 密钥生成阶段检测并拒绝               |
| d < 0                | ext_gcd 返回负 x       | gen_keys 中调整 d = d + φ | 模逆元取最小非负剩余                 |
| exp = 0              | fast_pow(base, 0, mod) | 返回 1 % mod              | 任何数的 0 次幂为 1                  |
| is_prime(1)          | 输入为 1               | 返回 0 (非素数)           | 小情况处理：num < 2 → false          |
| is_prime(2)          | 输入为 2               | 返回 1 (素数)             | 唯一的偶素数                         |

## 7. 常见错误

| 错误                                     | 后果                     | 正确做法                                                |
| ---------------------------------------- | ------------------------ | ------------------------------------------------------- |
| `ext_gcd` 中忘记递归基准 `b==0`          | 无限递归，栈溢出         | 首先检查 `if (b == 0) { *px=1; *py=0; return a; }`      |
| 私钥 d 直接使用负的 x 值                 | 解密结果不正确           | `while (d < 0) d += phi; d = d % phi;`                  |
| `fast_pow` 中忘记 `base = base % mod`    | 首次平方可能溢出         | 循环前执行 `base = base % mod`                          |
| `fast_pow` 中每次乘法后未取模            | 中间结果溢出 uint64_t    | 每次乘法后立即 `% mod`                                  |
| printf 格式用 `%lu` 而非 `PRIu64`        | 32 位平台输出错误        | 使用 `"%" PRIu64` 宏                                    |
| printf 中 `PRIu64` 写作 `"PRIu64"`       | 字面输出 "PRIu64"        | 正确写法：`"%" PRIu64`（注意引号位置）                  |
| `ext_gcd` 中 a/b 整数除法方向            | y 值错误                 | 使用 `a / b`（C 的整数除法向零取整）                    |
| 忘记 `#include <inttypes.h>`             | PRIu64 未定义            | 添加 `#include <inttypes.h>`                            |
| `is_prime` 未处理偶数                    | 性能差（仍能工作）       | 开头检查 `if ((num & 1) == 0) return num == 2;`         |
| `is_prime` 见证人 a >= num 时未跳过      | 可能错误判定（如 num=3） | `if (a >= num) continue;`                               |
| `gen_keys` 中未验证 e 的范围             | 接受非法 e=0 或 e=φ      | 检查 `e > 1 && e < phi`                                 |

## 8. 重要知识点

- **非对称加密的核心思想**: 加密和解密使用不同密钥。公钥可公开分发，
  私钥必须保密。数学上从公钥推导私钥是计算困难的。

- **RSA 安全性依赖于大整数分解困难性**: 已知 n = p*q，无法高效恢复 p 和 q。
  本题 n=3233 是玩具规模（可被立即分解），真实 RSA 使用 ≥2048 位的 n。

- **Miller-Rabin 是工业标准的素性测试**: OpenSSL、GnuPG、Java BigInteger
  都使用 Miller-Rabin 进行素性验证。它速度快、实现简单、错误概率可控。

- **欧拉定理是 RSA 正确性的基石**: a^φ(n) ≡ 1 (mod n)，使得加密和解密互为逆运算。
  前提是 gcd(m, n) = 1，这在实践中以极高概率成立。

- **扩展欧几里得算法是密码学的基本工具**: 不仅用于 RSA 密钥生成，
  也用于 ECC 点运算、中国剩余定理加速、以及许多零知识证明系统。

- **快速幂取模是密码学实现的必备技巧**: 将 O(exp) 降到 O(log exp)，
  使得 d=2753 这样的指数也能高效计算。实际 RSA 的 d 有 2048 位，
  不使用快速幂则完全不可行。

- **模运算的分配律保证中间结果不溢出**: (a * b) % m = ((a % m) * (b % m)) % m。
  每次乘法后取模，结果始终在 [0, m-1] 范围内。

- **密钥生成流程体现防御式编程**: gen_keys 不仅要计算，更要验证——检查 e 的范围、
  验证 gcd(e,φ)==1、验证 e*d mod φ == 1。密码学代码中，验证永远不嫌多。

- **uint64_t 玩具版的限制**: 本题 n=3233 很小，所有计算在 uint64_t 内安全。
  真实 RSA 需要大数库（如 GMP、OpenSSL 的 BIGNUM），因为 n 有数百位十进制数字。

- **Miller-Rabin 见证人的选择是经过数学证明的**: {2, 3, 5, 7, 11} 这 5 个见证人
  对所有 < 2^64 的数给出确定性结果——这不是巧合，而是经过严格的数学推导和验证。

## 9. 课堂讨论

**Q1: 为什么公钥指数 e 必须与 φ(n) 互质？**

A: 解密需要 e 模 φ(n) 的乘法逆元 d（即 e*d ≡ 1 mod φ(n)）。
整数 a 模 m 存在乘法逆元当且仅当 gcd(a, m) = 1。
如果 gcd(e, φ) ≠ 1，则不存在 d，解密将无法进行。
实际上常用 e=65537（素数，与任何 φ 几乎总互质），因为它是 2^16+1，
二进制只有两个 1，加密运算特别快。

**Q2: Miller-Rabin 是概率算法，为什么本题可以给出确定性结果？**

A: Miller-Rabin 本身是概率算法：对 k 个随机见证人，错误概率 < 4^(-k)。
但通过选择特定见证人集合，可以对特定范围给出确定性结果。
Gerhard Jaeschke (1993) 和 Jim Sinclair (2011) 的研究证明了
{2, 3, 5, 7, 11} 这 5 个见证人对所有 n < 2^64 给出确定性结果。
因为 uint64_t 的最大值约 1.8×10^19 < 2^64，所以在这个范围内是确定性的。

**Q3: 如果有人截获了密文 c=2557 和公钥 (e=17, n=3233)，他们能恢复明文吗？**

A: 本题可以，因为 n=3233 太小，可以轻易分解出 p=61, q=53，
从而计算 φ=3120 和 d=2753。真实 RSA 的 n 有 2048 位（约 617 个十进制位），
即使使用最先进的算法和超级计算机，分解也需要数十亿年。

**Q4: 扩展欧几里得算法中 x 可能是负数，如何处理？**

A: 负的 x 加上 φ 的倍数后仍是合法的模逆元（因为同余类等价）。
通常取最小非负剩余：`d = (x % φ + φ) % φ`。
本题中 x=-367，d = -367 + 3120 = 2753。

**Q5: 快速幂算法为什么是 O(log exp) 而不是 O(exp)？**

A: 快速幂通过"平方 - 乘"策略将问题规模每次减半。指数的二进制表示
有 log₂(exp) 位，每一位对应一次平方操作（最多 log 次），
每一位为 1 时额外一次乘法（平均 log/2 次）。总计约 1.5*log₂(exp) 次乘法，
而非 exp 次。exp=2753 时，朴素法需要 2753 次，快速幂只需约 18 次。

**Q6: 本题的 RSA 实现是"教科书式 RSA"，它有哪些安全缺陷？**

A: 教科书式 RSA 存在多个安全问题：(1) 确定性加密——相同明文产生相同密文，
容易泄露信息；(2) 对选择密文攻击（CCA）不安全；(3) 短消息容易被穷举；
(4) 没有随机性，语义不安全；(5) 没有填充，可能被小指数攻击（当 m^e < n 时）。
实际应用必须使用 OAEP 等填充方案。

**Q7: gen_keys 中为什么要验证 e 的范围 (1 < e < φ)？**

A: 如果 e = 1，则加密不改变消息（c = m^1 mod n = m），完全没有安全性。
如果 e = φ 或 e = 0，则加密也不安全。而且 e 必须与 φ 互质，
当 e ≥ φ 时可以通过模运算化简 e' = e mod φ（因为欧拉定理保证
m^e ≡ m^(e mod φ) mod n），所以 e 的范围在 [2, φ-1] 就足够了。
防御式编程要求明确检查这些边界条件。
