## Lesson 63 — 矩阵 LU 分解求解器（带部分主元）

### 课程任务

### 前置知识

#### 线性方程组的标准解法

对于方程组 Ax = b，大学线性代数课通常教三种方法：

```
┌─────────────────────────────────────────────────────────────┐
│ 方法           │ 复杂度    │ 数值稳定性 │ 实际使用        │
├─────────────────────────────────────────────────────────────┤
│ Cramer法则      │ O(n!)    │ 差         │ 仅理论/小矩阵    │
│ 求逆矩阵 A⁻¹    │ O(n³)    │ 差         │ 几乎不用         │
│ 高斯消元        │ O(n³)    │ 中         │ 教学常用         │
│ LU分解+主元     │ O(n³)    │ 好         │ 工业标准 ★      │
└─────────────────────────────────────────────────────────────┘
```

LU 分解的优势：

- 如果 A 固定而 b 变化多次，只需分解一次，每次求解只需 O(n²)
- 可以方便地计算行列式 det(A) = det(U)（注意置换符号）
- 可以求逆矩阵（解 n 次 Ly=Peⱼ, Ux=y）

#### 矩阵三角分解的直观理解

LU 分解本质上是高斯消元的矩阵形式：

```
高斯消元过程:
  A ──消去第1列──▶ A₁ ──消去第2列──▶ A₂ = U (上三角)

  ┌       ┐   l₂₁倍消去   ┌           ┐   l₃₂倍消去   ┌           ┐
  │ 2 1 1 │  ─────────▶  │ 2   1   1 │  ─────────▶  │ 2   1   1 │
  │ 4 3 3 │              │ 0   1   1 │              │ 0   1   1 │
  │ 8 7 9 │              │ 0   3   5 │              │ 0   0   2 │
  └       ┘              └           ┘              └           ┘
     A                       A₁                         A₂ = U

消元乘数组成 L:
  l₂₁ = 4/2 = 2
  l₃₁ = 8/2 = 4
  l₃₂ = 3/1 = 3

  ┌       ┐   ┌       ┐ ┌       ┐
  │ 1 0 0 │   │ 2 1 1 │ │ 2 1 1 │
  │ 2 1 0 │ × │ 0 1 1 │=│ 4 3 3 │ = A  ✓
  │ 4 3 1 │   │ 0 0 2 │ │ 8 7 9 │
  └       ┘   └       ┘ └       ┘
     L            U         A
```

#### 为什么要选主元（Pivoting）

不选主元时，小主元会导致灾难性后果：

```
示例:  ε = 0.0001（接近0的小数）

  [ε  1] [x₁] = [1]
  [1  1] [x₂]   [2]

不选主元的高斯消元:
  l₂₁ = 1/ε = 10000
  row₂ -= 10000·row₁
  → 0·x₁ + (1-10000)·x₂ = 2-10000
  → -9999·x₂ = -9998
  → x₂ ≈ 0.9999

  x₁ = (1 - 1·x₂)/ε = (1 - 0.9999)/0.0001 = 1.0

结果似乎 OK？但考虑浮点精度:
  -9999·x₂ = -9998  在 3 位精度下: -10000·x₂ ≈ -10000
  → x₂ = 1.0
  → x₁ = (1-1)/0.0001 = 0  ← 完全错误！

选主元后:
  交换两行:
  [1  1] [x₁] = [2]
  [ε  1] [x₂]   [1]

  l₂₁ = ε/1 = ε
  row₂ -= ε·row₁
  → (1-ε)·x₂ = 1-2ε
  → x₂ ≈ 1.0
  → x₁ = 2 - 1·1 = 1.0  ← 正确！
```

部分主元（Partial Pivoting）策略：

- 第 k 步时，在第 k 列中寻找 |A[i][k]| 最大的行
- 将该行交换到第 k 行
- 保证所有乘数 |l_ik| ≤ 1，避免误差放大

```
选主元示意（本题 k=0）:

  列0:  A[0][0]=2, A[1][0]=4, A[2][0]=8
        max |·| = 8 在第2行
        → 交换 row0 ↔ row2

  交换前:              交换后:
  ┌       ┐           ┌       ┐
  │ 2 1 1 │  row0     │ 8 7 9 │  ← 来自原row2
  │ 4 3 3 │  row1     │ 4 3 3 │
  │ 8 7 9 │  row2     │ 2 1 1 │  ← 来自原row0
  └       ┘           └       ┘
```

#### 排列向量的作用

排列向量 p 记录了行交换的历史：

```
初始: p = [0, 1, 2]  (恒等排列)

k=0: swap row0↔row2 → p = [2, 1, 0]
     (p[0]=2 表示"第0行现在存的是原来的第2行")

k=1: swap row1↔row2 → p = [2, 0, 1]
     (p[0]=2, p[1]=0, p[2]=1)

解读: 最终第0行=原第2行, 第1行=原第0行, 第2行=原第1行
```

前向替换时需要将 b 按同样方式排列：

```
Pb = [b[p[0]], b[p[1]], b[p[2]]]
   = [b[2], b[0], b[1]]
   = [29, 5, 11]
```

### 算法详解

#### 步骤一：PA = LU 分解

```
算法: LU分解 with Partial Pivoting (In-Place)

输入: A (N×N矩阵)
输出: A 被覆盖为 L+U (不含L对角线), p (排列向量)

1. for i = 0 to N-1:
      p[i] = i                    // 初始化排列为恒等

2. for k = 0 to N-1:
      // 部分主元
      max_row = k
      for i = k+1 to N-1:
          if |A[i][k]| > |A[max_row][k]|:
              max_row = i
      if max_row != k:
          swap row k ↔ row max_row in A
          swap p[k] ↔ p[max_row]
          print pivot info

      // 消去
      for i = k+1 to N-1:
          A[i][k] = A[i][k] / A[k][k]        // 乘数 l_ik
          for j = k+1 to N-1:
              A[i][j] = A[i][j] - A[i][k] * A[k][j]

3. 提取 L 和 U:
      for i, j = 0 to N-1:
          if i > j:  L[i][j] = A[i][j], U[i][j] = 0
          if i == j: L[i][j] = 1,       U[i][j] = A[i][j]
          if i < j:  L[i][j] = 0,       U[i][j] = A[i][j]
```

In-Place 存储示意（分解后的 A）：

```
A (分解后):
┌─────────────────────┐
│ u₀₀     u₀₁    u₀₂ │  ← U 的上三角（含对角线）
│ l₁₀     u₁₁    u₁₂ │
│ l₂₀     l₂₁    u₂₂ │  ← L 的严格下三角（对角线为1，不存储）
└─────────────────────┘
```

本题的详细计算过程：

```
初始 A = [[2, 1, 1],
          [4, 3, 3],
          [8, 7, 9]]

═══════════════════════════════════════
k=0: 选主元
═══════════════════════════════════════

列0: |A[0][0]|=2, |A[1][0]|=4, |A[2][0]|=8
max = 8 at row 2 → swap row0 ↔ row2

A = [[8, 7, 9],     p = [2, 1, 0]
     [4, 3, 3],
     [2, 1, 1]]

消去:
  i=1: l₁₀ = 4/8 = 0.5
       A[1][1] = 3 - 0.5*7 = -0.5
       A[1][2] = 3 - 0.5*9 = -1.5
  i=2: l₂₀ = 2/8 = 0.25
       A[2][1] = 1 - 0.25*7 = -0.75
       A[2][2] = 1 - 0.25*9 = -1.25

A = [[8,   7,     9   ],
     [0.5, -0.5,  -1.5],
     [0.25,-0.75, -1.25]]

═══════════════════════════════════════
k=1: 选主元
═══════════════════════════════════════

列1 (i≥1): |A[1][1]|=0.5, |A[2][1]|=0.75
max = 0.75 at row 2 → swap row1 ↔ row2

A = [[8,   7,     9   ],     p = [2, 0, 1]
     [0.25,-0.75, -1.25],
     [0.5, -0.5,  -1.5 ]]

消去:
  i=2: l₂₁ = -0.5/(-0.75) = 0.6667
       A[2][2] = -1.5 - 0.6667*(-1.25) = -1.5 + 0.8333 = -0.6667

A = [[8,   7,      9    ],
     [0.25,-0.75,  -1.25],
     [0.5,  0.6667,-0.6667]]

═══════════════════════════════════════
提取 L 和 U
═══════════════════════════════════════

L = [[1,     0,      0    ],    U = [[8,   7,      9    ],
     [0.25,  1,      0    ],        [0,  -0.75,  -1.25],
     [0.5,   0.6667, 1    ]]        [0,   0,     -0.6667]]

验证 PA = LU:
  P = [[0,0,1],    PA = [[8,7,9],
       [1,0,0],          [2,1,1],
       [0,1,0]]          [4,3,3]]

  L*U row0 = [8, 7, 9] ✓
  L*U row1 = [0.25*8, 0.25*7+(-0.75), 0.25*9+(-1.25)]
           = [2, 1, 1] ✓
  L*U row2 = [0.5*8, 0.5*7+0.6667*(-0.75), 0.5*9+0.6667*(-1.25)+(-0.6667)]
           = [4, 3.5-0.5, 4.5-0.8333-0.6667]
           = [4, 3, 3] ✓
```

#### 步骤二：前向替换 Ly = Pb

```
L = [[1,     0,      0    ],    Pb = [29, 5, 11]
     [0.25,  1,      0    ],
     [0.5,   0.6667, 1    ]]

y₀ = 29                              (L[0][0]=1, 无需除法)
y₁ = 5 - 0.25×29 = -2.25
y₂ = 11 - 0.5×29 - 0.6667×(-2.25)
   = 11 - 14.5 + 1.5
   = -2.0

y = [29, -2.25, -2.0]
```

前向替换的计算模式：

```
  y₀ = Pb₀                          ← 只依赖 Pb₀
  y₁ = Pb₁ - L[1][0]·y₀             ← 依赖 y₀
  y₂ = Pb₂ - L[2][0]·y₀ - L[2][1]·y₁ ← 依赖 y₀, y₁
```

#### 步骤三：后向替换 Ux = y

```
U = [[8,   7,      9    ],    y = [29, -2.25, -2.0]
     [0,  -0.75,  -1.25],
     [0,   0,     -0.6667]]

x₂ = -2.0 / (-0.6667) = 3.0
x₁ = (-2.25 - (-1.25)×3.0) / (-0.75)
   = (-2.25 + 3.75) / (-0.75)
   = 1.5 / (-0.75) = -2.0
x₀ = (29 - 7×(-2.0) - 9×3.0) / 8
   = (29 + 14 - 27) / 8
   = 16 / 8 = 2.0

x = [2.0, -2.0, 3.0]
```

后向替换的计算模式：

```
  x₂ = y₂ / U[2][2]                              ← 只依赖 y₂
  x₁ = (y₁ - U[1][2]·x₂) / U[1][1]              ← 依赖 x₂
  x₀ = (y₀ - U[0][1]·x₁ - U[0][2]·x₂) / U[0][0] ← 依赖 x₁, x₂
```

#### 步骤四：验证 Ax = b

```
Ax = [[2,1,1],   × [ 2]   = [2×2 + 1×(-2) + 1×3]   = [4-2+3]     = [ 5] ✓
      [4,3,3],     [-2]     [4×2 + 3×(-2) + 3×3]     [8-6+9]       [11] ✓
      [8,7,9]]     [ 3]     [8×2 + 7×(-2) + 9×3]     [16-14+27]    [29] ✓
```

### 知识点

#### 1. 三角矩阵求解的效率

一般线性方程组求解是 O(N³)，但三角矩阵可以 O(N²) 求解：

```
┌────────────────────────────────────────────────────┐
│ 操作         │ 一般矩阵  │ 三角矩阵   │ 加速比(N=3) │
├────────────────────────────────────────────────────┤
│ 前向替换      │ O(N³)    │ O(N²)     │ ~3x         │
│ 后向替换      │ O(N³)    │ O(N²)     │ ~3x         │
│ LU分解        │ -        │ O(N³)     │ -           │
│ 总计(一次求解) │ O(N³)    │ O(N³)     │ 相当         │
│ 总计(m次求解)  │ m·O(N³)  │ O(N³)+m·O(N²) │ ~m倍   │
└────────────────────────────────────────────────────┘
```

当需要求解多个右端项 b 时，LU 分解只需做一次 O(N³) 分解，
之后每个 b 的求解只需 O(N²)。

#### 2. 浮点精度与数值稳定性

LU 分解涉及大量浮点运算，精度问题不可忽视：

```
浮点误差来源:
  ┌─────────────────────────────────────────────┐
  │ 1. 舍入误差   │ 有限位数无法精确表示某些数  │
  │ 2. 抵消误差   │ 相近的大数相减损失有效位数  │
  │ 3. 放大误差   │ 大乘数放大之前步骤的误差    │
  └─────────────────────────────────────────────┘

部分主元的作用:
  - 保证 |l_ik| ≤ 1，抑制误差放大
  - 将大元素放在对角线上，减少抵消

使用 double (64位) 的精度:
  - 有效位数约 15-17 位十进制
  - 本题 N=3，条件数不大，double 绰绰有余
  - 对于病态矩阵（如 Hilbert 矩阵），即使选主元也可能不精确
```

精度验证示例：

```c
/* 验证 Ax 与 b 的接近程度 */
double error = 0.0;
for (int i = 0; i < N; i++) {
    double diff = Ax[i] - b[i];
    error += diff * diff;
}
printf("Residual ||Ax-b||² = %e\n", error);
/* 对于本题，残差应该在 1e-30 量级（机器精度内） */
```

#### 3. 排列矩阵与排列向量

排列矩阵 P 是正交矩阵（P⁻¹ = Pᵀ），每行每列恰好一个 1。

```
p = [2, 0, 1] 对应的排列矩阵:

      ┌       ┐
      │ 0 0 1 │  ← 第0行: 1在第2列 (取原第2行)
  P = │ 1 0 0 │  ← 第1行: 1在第0列 (取原第0行)
      │ 0 1 0 │  ← 第2行: 1在第1列 (取原第1行)
      └       ┘

  PA = P × A: 用 P 的行排列 A 的行
  AP = A × P: 用 P 的列排列 A 的列

  验证: P × [b₀,b₁,b₂]ᵀ = [b₂,b₀,b₁]ᵀ = Pb
```

#### 4. 行列式计算

LU 分解可以方便地计算行列式：

```
det(A) = det(P⁻¹) × det(L) × det(U)
       = det(P) × 1 × Π U[i][i]
       = (-1)^(交换次数) × Π U[i][i]

本题: 2次行交换 → det(P) = (-1)² = 1
     U对角线乘积 = 8 × (-0.75) × (-0.6667) = 4.0
     det(A) = 1 × 4.0 = 4.0

验证: |2 1 1|
      |4 3 3| = 2(27-21) - 1(36-24) + 1(28-24)
      |8 7 9| = 12 - 12 + 4 = 4 ✓
```

### 讨论问题

1. **如果不选主元会怎样？** 用本题的矩阵尝试不选主元的 LU 分解（总是用 A[k][k] 作为主元）。结果是否仍然正确？为什么？

2. **全主元 vs 部分主元：** 全主元（complete pivoting）同时搜索行和列中的最大元素，比部分主元更稳定但开销更大。在什么场景下需要全主元？

3. **稀疏矩阵的 LU 分解：** 如果 A 是稀疏矩阵（大部分元素为 0），标准 LU 分解会产生 fill-in（零元素变为非零）。如何利用稀疏结构优化？

4. **分块 LU 分解：** 对于大规模矩阵（N > 1000），现代 BLAS/LAPACK 使用分块算法利用 CPU 缓存。分块 LU 分解的基本思想是什么？

5. **Cholesky 分解：** 对于对称正定矩阵，Cholesky 分解 A = LLᵀ 比 LU 分解快一倍且不需要选主元。为什么本题不使用 Cholesky 分解？

6. **浮点精度：** 将 `double` 改为 `float` 后重新运行。Ax 与 b 的差异是否仍然在可接受范围内？在什么情况下 `float` 不够用？

### 常见错误排查

| 症状                                   | 可能原因               | 检查方法                   |
| -------------------------------------- | ---------------------- | -------------------------- |
| 编译错误 `undefined reference to fabs` | 忘记 `-lm`             | 添加 `LDLIBS = -lm`        |
| 输出全为 0                             | L/U 提取逻辑错误       | 检查 i>j, i==j, i<j 分支   |
| x 不正确但 L/U 正确                    | 前向替换忘记排列 b     | 确认 `Pb[i] = b[p[i]]`     |
| 前几步正确后几步错误                   | 后向替换忘记除 U[i][i] | 检查 `x[i] /= U[i][i]`     |
| pivot 未触发                           | max_val 初始化为 0     | 应初始化为 `fabs(A[k][k])` |
| L/U 输出混乱                           | 提取时行列索引搞反     | 确认 i 是行 j 是列         |
| `-nan` 出现在输出                      | 除以零                 | 检查 pivot 是否正常交换    |
