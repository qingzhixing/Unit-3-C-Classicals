### 前置知识：LL(1) 解析原理

#### 什么是 LL(1)

LL(1) 是自顶向下（top-down）语法分析的经典算法，名字的含义是：

```
  LL(1) = L  eft-to-right scan of input    （从左到右扫描输入）
         + L  eftmost derivation            （最左推导）
         + 1  lookahead token               （1 个前瞻符号）
```

核心思想：**每一步根据栈顶的语法符号和当前输入符号，查表决定使用哪条产生式**。

#### LL(1) 的三个关键数据结构

```
┌──────────────────────────────────────────────────────────┐
│                    LL(1) 解析器架构                        │
├──────────────────────────────────────────────────────────┤
│                                                          │
│   输入缓冲区                  栈（LIFO）                   │
│  ┌───┬───┬───┬───┬───┬───┐  ┌───────┐                   │
│  │id │ + │id │ * │id │ $ │  │   E   │ ← 栈顶              │
│  └───┴───┴───┴───┴───┴───┘  │   $   │ ← 栈底              │
│         ↑                    └───────┘                   │
│    前瞻指针 (ip)                                          │
│                                                          │
│              预测分析表 table[NT][T]                       │
│              ┌─────┬─────┬─────┬─────┐                    │
│              │ id  │  +  │  *  │ ... │                    │
│         ┌────┼─────┼─────┼─────┼─────┤                    │
│         │ E  │  0  │  -  │  -  │ ... │                    │
│         │ E' │  -  │  1  │  -  │ ... │                    │
│         │ T  │  3  │  -  │  -  │ ... │                    │
│         └────┴─────┴─────┴─────┴─────┘                    │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

#### 栈式解析的执行循环

```
while (栈非空):
    X = 栈顶符号
    a = 当前输入符号

    if X 是终结符:
        if X == a:  弹出 X，ip++（匹配成功）
        else:       ERROR（终结符不匹配）

    if X 是非终结符:
        p = table[X][a]    ← 查预测分析表
        if p == -1:  ERROR（语法错误，表中无对应产生式）
        else:        弹出 X，按产生式 p 的 RHS 压栈（从右到左）
```

### 文法与产生式

本练习使用的表达式文法（消除左递归后）：

```
E  → T E'          (产生式 0)    ← 开始符号
E' → + T E'        (产生式 1)
E' → ε             (产生式 2)
T  → F T'          (产生式 3)
T' → * F T'        (产生式 5)
T' → ε             (产生式 4)
F  → ( E )         (产生式 7)
F  → id            (产生式 6)
```

这是一个经典的表达式文法，处理 `+` 和 `*` 两种运算符，`*` 优先级高于 `+`。

#### 为什么需要 E' 和 T'

```
原始文法（含左递归）：         消除左递归后：
E → E + T | T                E  → T E'
T → T * F | F                E' → + T E' | ε
F → ( E ) | id               T  → F T'
                             T' → * F T' | ε
                             F  → ( E ) | id
```

左递归文法会导致递归下降解析器无限递归。LL(1) 要求文法无左递归、无公共前缀。

### 算法详解：FIRST 集计算

#### 定义

FIRST(α) = { a | α ⇒* aβ, a 是终结符 } ∪ { ε | 若 α ⇒* ε }

即：从符号串 α 能推导出的所有首终结符的集合。

#### 不动点迭代算法

```
初始化：first[NT] = ∅（所有位掩码为 0）

重复直到收敛：
  for 每个产生式 A → X1 X2 ... Xn:
    for i = 1 to n:
      if Xi 是终结符 a:
        first[A] = first[A] ∪ {a}
        break                     ← 终结符不可空，停止
      else (Xi 是非终结符):
        first[A] = first[A] ∪ (first[Xi] - {ε})
        if Xi 不可空:  break      ← Xi 不能推导出 ε，停止
        /* Xi 可空则继续看下一个符号 */
```

#### 本练习中哪些符号可空

| 非终结符 | 可空？ | 原因                          |
| -------- | ------ | ----------------------------- |
| E        | 否     | E→TE'，T 不可空               |
| E'       | 是     | E'→ε 产生式                   |
| T        | 否     | T→FT'，F 不可空               |
| T'       | 是     | T'→ε 产生式                   |
| F        | 否     | F→id 或 F→(E)，均以终结符开头 |

#### 位掩码表示

每个 FIRST 集用一个 `int` 位掩码表示：

```
bit 0 → id    bit 1 → +    bit 2 → *
bit 3 → (     bit 4 → )    bit 5 → $
```

例如 `FIRST(F) = {id, (}` → 位掩码 = (1<<0) | (1<<3) = 1 + 8 = 9。

```
位掩码操作宏：
  HAS(mask, t)  →  (mask) & (1 << (t))    检查 bit t 是否为 1
  ADD(mask, t)  →  (mask) |= (1 << (t))   将 bit t 置 1
```

#### 迭代过程示例

```
初始：first[E]=∅, first[E']=∅, first[T]=∅, first[T']=∅, first[F]=∅

第1轮：
  产生式 0 (E→TE'):   X1=T(非终结符), first[T]=∅, T不可空 → first[E]=∅
  产生式 6 (F→id):    X1=id(终结符) → first[F]={id}
  产生式 7 (F→(E)):   X1=( → first[F]={id,(}

第2轮：
  产生式 3 (T→FT'):   X1=F, first[F]={id,(}, F不可空 → first[T]={id,(}
  产生式 0 (E→TE'):   X1=T, first[T]={id,(}, T不可空 → first[E]={id,(}

第3轮：
  产生式 1 (E'→+TE'): X1=+(终结符) → first[E']={+}
  产生式 5 (T'→*FT'): X1=* → first[T']={*}

第4轮：无变化 → 收敛
```

### 算法详解：FOLLOW 集计算

#### 定义

FOLLOW(A) = { a | S ⇒* ...Aa..., a 是终结符 }

即：在某个句型中，紧跟在非终结符 A 后面的所有终结符的集合。

#### 不动点迭代算法

```
初始化：follow[NT] = ∅
follow[开始符号] = follow[开始符号] ∪ {$}

重复直到收敛：
  for 每个产生式 A → αBβ:
    /* 规则1: FOLLOW(B) ⊇ FIRST(β) - {ε} */
    for 每个终结符 a ∈ FIRST(β):
      follow[B] = follow[B] ∪ {a}

    /* 规则2: 若 β ⇒* ε，则 FOLLOW(B) ⊇ FOLLOW(A) */
    if β ⇒* ε:
      follow[B] = follow[B] ∪ follow[A]
```

#### 迭代过程示例

```
初始：follow[E]={$}, 其他=∅

第1轮：
  产生式 0 (E→TE'):
    B=T, β=E': first[E']={+}, E'可空
      → follow[T] ∪= {+}         (规则1)
      → follow[T] ∪= follow[E]   (规则2, β可空)
      → follow[T]={+,$}
    B=E', β=∅ (空):
      → follow[E'] ∪= follow[E]  (规则2)
      → follow[E']={$}

  产生式 1 (E'→+TE'):
    B=T, β=E': first[E']={+}, E'可空
      → follow[T] ∪= {+}
      → follow[T] ∪= follow[E']
      → follow[T]={+,$}
    B=E', β=∅:
      → follow[E'] ∪= follow[E'] (自反，无变化)

  ... 继续迭代直到收敛
```

#### 最终结果

| 非终结符 | FIRST   | FOLLOW       |
| -------- | ------- | ------------ |
| E        | {id, (} | {), $}       |
| E'       | {+}     | {), $}       |
| T        | {id, (} | {+, ), $}    |
| T'       | {*}     | {+, ), $}    |
| F        | {id, (} | {+, *, ), $} |

### 算法详解：预测分析表构造

#### 构造规则

对每个产生式 `A → α`：

```
规则1: 对每个终结符 a ∈ FIRST(α)，table[A][a] = 产生式编号
规则2: 若 ε ∈ FIRST(α)（即 α ⇒* ε），
       则对每个终结符 b ∈ FOLLOW(A)，table[A][b] = 产生式编号
```

#### 构造过程示例

```
产生式 0 (E→TE'): FIRST(TE')={id,(}
  → table[E][id]=0, table[E][(]=0

产生式 2 (E'→ε): ε∈FIRST(ε), FOLLOW(E')={),$}
  → table[E'][)]=2, table[E'][$]=2

产生式 1 (E'→+TE'): FIRST(+TE')={+}
  → table[E'][+]=1

产生式 4 (T'→ε): FOLLOW(T')={+,),$}
  → table[T'][+]=4, table[T'][)]=4, table[T'][$]=4

产生式 5 (T'→*FT'): FIRST(*FT')={*}
  → table[T'][*]=5

... 依此类推
```

#### 完整预测分析表

|     | id  | +   | *   | (   | )   | $   |
| --- | --- | --- | --- | --- | --- | --- |
| E   | 0   | —   | —   | 0   | —   | —   |
| E'  | —   | 1   | —   | —   | 2   | 2   |
| T   | 3   | —   | —   | 3   | —   | —   |
| T'  | —   | 4   | 5   | —   | 4   | 4   |
| F   | 6   | —   | —   | 7   | —   | —   |

数字 = 产生式编号，— = -1（语法错误）

### RHS 编码方案

RHS 数组中，终结符和非终结符使用不同编码区分：

```
值范围       含义                 示例
─────────   ──────────────────   ────────────
0 ~ 5       终结符枚举值          T_ID(0), T_PLUS(1), ...
100 ~ 104   非终结符(NT+100)      NT(NT_E)=100, NT(NT_T)=102
-1          结束标记              序列终止
```

**关键：rhs 数组是逆序存储的**——最右侧符号在索引 0，最左侧符号在最后。

```
示例：E' → + T E' 的 RHS = {NT_E'+100, NT_T+100, T_PLUS, -1}
                            = {101, 102, 1, -1}

栈压入顺序（i=0 到 -1）:
  i=0: 101 → push(NONTERM, 1) = E'     ← 最右侧符号先压入
  i=1: 102 → push(NONTERM, 2) = T
  i=2: 1   → push(TERM, 1)     = +

压完后栈顶是 '+'（最先匹配），然后是 T，最后是 E'。
这正是 LIFO 特性：后压入的先出来。
```

**正向读取 RHS 符号使用 `rhs_at(p, i)` 函数**：

```c
/* 获取产生式 p 从左到右的第 i 个符号（0 = 第一个符号） */
static int rhs_at(int p, int i) {
    int len = rhs_len(p);
    return rhs[p][len - 1 - i];  /* 索引映射 */
}
```

### 解析过程完整追踪

输入 `id + id * id $`，以下是每一步的栈、输入和动作：

```
Step   栈（栈底→栈顶）         输入（ip→末尾）        动作
────   ─────────────────────   ────────────────────   ────────────
[00]   $E                       id+id*id$              E  → T E'
[01]   $E'T                     id+id*id$              T  → F T'
[02]   $E'T'F                   id+id*id$              F  → id
[03]   $E'T'id                  id+id*id$              match id
[04]   $E'T'                    +id*id$                T' → ε
[05]   $E'                      +id*id$                E' → + T E'
[06]   $E'T+                    +id*id$                match +
[07]   $E'T                     id*id$                 T  → F T'
[08]   $E'T'F                   id*id$                 F  → id
[09]   $E'T'id                  id*id$                 match id
[10]   $E'T'                    *id$                   T' → * F T'
[11]   $E'T'F*                  *id$                   match *
[12]   $E'T'F                   id$                    F  → id
[13]   $E'T'id                  id$                    match id
[14]   $E'T'                    $                      T' → ε
[15]   $E'                      $                      E' → ε
[16]   $                        $                      match $
[17]                                                    ACCEPT ✓
```

#### 关键步骤详解

**Step 0: 初始状态**

```
栈: $ E        （栈底 $，栈顶 E）
输入: id + id * id $
查表: table[E][id] = 0 → E → T E'
动作: pop E, push E', push T
结果: 栈 = $ E' T
```

**Step 4: T' → ε（空产生式）**

```
栈: $ E' T'    （栈顶 T'）
输入: + id * id $
查表: table[T'][+] = 4 → T' → ε
动作: pop T'，不压入任何符号（ε 产生式）
结果: 栈 = $ E'
```

**Step 16: 匹配 $**

```
栈: $           （栈顶 $）
输入: $
动作: match $，pop $，ip++
结果: 栈为空，循环结束
```

### 边缘情况分析

| 场景                     | 预期行为                                    | 原因                                |
| ------------------------ | ------------------------------------------- | ----------------------------------- |
| 输入为空（只有 $）       | 查表失败 → ERROR                            | E 不能推导出 ε，table[E][$] = -1    |
| 括号不匹配 `( id $`      | 匹配 `(` 后栈中有 `)`，但输入是 `$` → ERROR | table[E][$] = -1 或终结符不匹配     |
| 连续运算符 `id + + id`   | 第一个 `+` 匹配后，栈顶 E' 查 `+` 产生式    | 正常展开，第二个 `+` 由内层 E' 处理 |
| 只有单个 id              | E→TE'→FT'→id, T'→ε, E'→ε → ACCEPT           | id 是合法的表达式                   |
| 嵌套括号 `((id))`        | F→(E) 递归展开，每层匹配一对括号            | 递归嵌套由栈自动处理                |
| 输入含未知符号 `id @ id` | token_id 返回 -1 或表中无对应产生式         | 词法错误或语法错误                  |

### 比较：LL(1) vs 递归下降

| 维度         | LL(1) 表驱动                     | 递归下降                          |
| ------------ | -------------------------------- | --------------------------------- |
| 实现方式     | 显式栈 + 预测分析表（二维数组）  | 函数递归调用栈 + 硬编码分支       |
| 文法变更响应 | 重新计算 FIRST/FOLLOW 和表       | 修改对应递归函数                  |
| 代码规模     | 通用解析循环 + 表初始化代码      | 每个非终结符一个函数              |
| 错误恢复     | 可在表中插入错误恢复产生式       | 需要在每个函数中手动处理          |
| 性能         | 查表 O(1)，但栈操作有开销        | 函数调用有栈帧开销，但无显式栈    |
| 学习价值     | 深入理解 FIRST/FOLLOW/预测分析表 | 直观理解递归下降和回溯            |
| 典型应用     | yacc/bison 生成的 LALR(1) 解析器 | 手写解析器（Clang, GCC 早期版本） |

### 常见错误与陷阱

| 错误                                      | 后果                                               | 正确做法                                                 |
| ----------------------------------------- | -------------------------------------------------- | -------------------------------------------------------- |
| rhs 数组正向读取而非用 rhs_at()           | FIRST/FOLLOW 计算用错符号顺序，结果错误            | 始终使用 `rhs_at(p, i)` 获取从左到右第 i 个符号          |
| FIRST 迭代中，终结符后忘记 break          | 继续处理后续符号，FIRST 集错误地包含不应有的终结符 | 终结符不可空，匹配后立即 `break`                         |
| FOLLOW 迭代中，β 全可空时未复制 FOLLOW(A) | FOLLOW 集缺失 $ 或 )，导致表项缺失                 | `beta_all_nullable` 为真时，执行 `FOLLOW(B) ⊇ FOLLOW(A)` |
| 可空判断硬编码了 E' 和 T'                 | 不通用，换文法就错                                 | 本练习中可空性是固定的（E' 和 T'），可以硬编码           |
| 构建表时忘记初始化 table[][] = -1         | 未填的位置是随机值，查表时产生不可预测的行为       | 先双重循环 `table[nt][t] = -1`                           |
| 空产生式（ε）忘记处理                     | 表中缺少 FOLLOW(A) 对应的表项                      | 检查 `rhs_len(p) == 0` 的情况                            |
| 压栈顺序错误（从右到左 vs 从左到右）      | 解析顺序反了，第一个符号被压到栈底                 | rhs 已逆序存储，直接 for i=0 遍历压栈即可                |
| set_str 打印格式不对                      | diff 失败                                          | 仔细比对 expected_output.txt：`{id, (}` 注意逗号后的空格 |

### 重要知识点

- **FIRST 集的本质**：告诉你"面对非终结符 A 时，下一个可能读到的终结符是什么"。如果 `a ∉ FIRST(A)`，那 `A → ...` 的产生式不可能匹配以 `a` 开头的输入。

- **FOLLOW 集的本质**：告诉你"非终结符 A 什么时候可以消失（推导出 ε）"。只有当前输入符号 `∈ FOLLOW(A)` 时，才能使用 `A → ε` 产生式。

- **位掩码的优势**：用 `int` 的 bit 表示集合成员，ADD/HAS 操作都是 O(1) 的位运算。本练习 6 个终结符刚好装进一个 `int`。

- **不动点迭代的通用性**：FIRST/FOLLOW 计算是经典的"不动点"（fixed-point）算法——不断应用规则直到没有新元素加入。这种方法对任何 CFG 都适用。

- **预测分析表的二义性检测**：如果构造过程中发现 `table[A][a]` 已经被填入不同的产生式，说明文法不是 LL(1) 的——存在冲突（first/first 或 first/follow 冲突）。

- **ε 产生式的特殊性**：`A → ε` 意味着 A 可以"消失"。在 FIRST 中不显式表示 ε（只用位掩码存终结符），但在 FOLLOW 中通过"β 可空"传播来处理。

- **表驱动解析的优雅之处**：解析逻辑（while 循环）与文法完全分离。文法信息全部编码在 table 数组中。要换一个 LL(1) 文法，只需重新计算表，解析循环不需要修改。

### 课堂讨论

1. **为什么叫 LL(1) 而不是 LL(k)？** 本题文法只需要 1 个前瞻符号即可确定产生式。某些文法（如 `if-then-else` 的悬空 else 问题）需要 k>1 的前瞻。LL(k) 表的大小随 k 指数增长，实际中很少用 k>1。

2. **如果文法不是 LL(1) 的会怎样？** 构建预测分析表时会出现"冲突"——同一个 `table[A][a]` 被两个不同的产生式填充。例如 `E → id | id ( E )`（两个产生式都以 `id` 开头），table[E][id] 冲突。解决方法：提取左因子或消除左递归。

3. **FIRST 迭代为什么一定能收敛？** 每个非终结符的 FIRST 集是有限集合（最多 T_COUNT 个元素），每次迭代只增不减（单调递增）。有限集上的单调递增序列必然在有限步内停止——这就是不动点定理的直觉。

4. **这个解析器能否扩展为表达式求值器？** 可以！在每个 reduce 动作（非终结符展开完成时）执行语义动作：`F → id` 时记录标识符的值，`T → T' * F T'` 展开完成时执行乘法，`E → E' + T E'` 展开完成时执行加法。这就是语法制导翻译（Syntax-Directed Translation）。

5. **为什么不直接用递归下降而要用表驱动？** 递归下降更直观，但文法改变时需要重写代码。表驱动解析器把文法信息数据化——FIRST/FOLLOW/table 就是数据，解析循环是通用引擎。这种"数据驱动"的设计是编译器自动生成工具（yacc/bison/ANTLR）的基础。

6. **位掩码表示的局限性是什么？** `int` 只有 32 或 64 位，最多表示 32/64 个终结符。对大型文法（如 C 语言有 100+ 关键字和运算符），需要用更复杂的数据结构（如 bitset 数组或链表）。但本练习的 6 个终结符完美适配位掩码。
