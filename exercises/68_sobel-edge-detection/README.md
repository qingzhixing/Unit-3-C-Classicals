## Lesson 68 — Sobel 边缘检测（8×8 PGM 图像含垂直/水平/对角边）【图像处理实战】

### 课程任务

用 C 语言对固定的 **8×8 灰度 PGM 图像**实现 **Sobel 边缘检测**。
图像包含**垂直边、水平边、对角边和均匀区域**四种边缘类型。

学员需要完成 5 个部分：

1. **`print_matrix()`** — 打印整数矩阵（带标题和格式对齐）
2. **`convolve()`** — 对图像像素执行 3×3 卷积
3. **`gradient_magnitude()`** — 用 Sobel 算子计算梯度幅值
4. **`threshold()`** — 阈值二值化（≥128 → 255, <128 → 0）
5. **`main()`** — 主流程：原始图像 → 梯度幅值 → 边缘二值图

Sobel 核 `GX`、`GY` 和图像数据 `IMAGE` 已完整提供，Sobel 核**保持不变**，学员只需实现算法流程。

### 前置知识：图像卷积

#### 什么是卷积？

卷积（Convolution）是图像处理中最基础的操作之一。它用一个小矩阵（称为"核"或"滤波器"）在图像上滑动，在每个位置计算核与图像局部区域的加权和。

```
滑动窗口示意图 (3×3 核在 8×8 图像上):

┌──────────────────────────────────────┐
│ I₀₀  I₀₁  I₀₂  I₀₃  I₀₄  I₀₅  I₀₆  I₀₇ │
│ I₁₀ ┌───────────────────┐ I₁₅  I₁₆  I₁₇ │
│ I₂₀ │ I₂₁  I₂₂  I₂₃  I₂₄ │ I₂₆  I₂₇ │  ← 核以 (2,2) 为中心
│ I₃₀ │ I₃₁  I₃₂  I₃₃  I₃₄ │ I₃₆  I₃₇ │    对 3×3 邻域加权求和
│ I₄₀ └───────────────────┘ I₄₆  I₄₇ │
│ I₅₀  I₅₁  I₅₂  I₅₃  I₅₄  I₅₅  I₅₆  I₅₇ │
│ I₆₀  I₆₁  I₆₂  I₆₃  I₆₄  I₆₅  I₆₆  I₆₇ │
│ I₇₀  I₇₁  I₇₂  I₇₃  I₇₄  I₇₅  I₇₆  I₇₇ │
└──────────────────────────────────────┘

对于中心像素 (row, col) 的卷积:
  sum = Σ(i=-1..1) Σ(j=-1..1) image[row+i][col+j] × kernel[i+1][j+1]
```

核在图像上逐像素滑动，每次计算一个输出值。对于 8×8 的图像，只有内部的 6×6 个像素有完整的 3×3 邻域（边界像素的邻域会越界，本题中设为 0）。

#### 为什么卷积能检测边缘？

边缘是图像中亮度急剧变化的地方。如果核的权重一边为正、一边为负，那么当它跨越一个亮度变化区域时，正负区域会产生很大的加权和（绝对值大）；而在均匀区域，正负相消，结果接近零。

### 前置知识：Sobel 算子

#### Sobel 核定义

Sobel 算子由两个 3×3 核组成，分别检测水平和垂直方向的边缘：

```
Gx (检测垂直边缘 — 水平方向梯度):
        ┌            ┐
        │ -1   0   1 │
        │ -2   0   2 │
        │ -1   0   1 │
        └            ┘

Gy (检测水平边缘 — 垂直方向梯度):
        ┌              ┐
        │ -1  -2  -1  │
        │  0   0   0  │
        │  1   2   1  │
        └              ┘
```

#### 核的直观理解

**Gx 核** — 检测垂直边缘：

- 右侧列权重为 +1, +2, +1（正值）
- 左侧列权重为 -1, -2, -1（负值）
- 中间列为 0（不关心）
- 当核跨越一条**垂直边缘**时（左侧暗、右侧亮），右侧正权重乘以亮值 + 左侧负权重乘以暗值 → 很大的正数
- 如果是水平边缘（上下差异），Gx 几乎为零

**Gy 核** — 检测水平边缘：

- 下方行权重为 +1, +2, +1（正值）
- 上方行权重为 -1, -2, -1（负值）
- 中间行为 0（不关心）
- 当核跨越一条**水平边缘**时（上方暗、下方亮），下方正权重乘以亮值 + 上方负权重乘以暗值 → 很大的正数
- 如果是垂直边缘（左右差异），Gy 几乎为零

#### 为什么中间行列权重加倍？

Sobel 算子给中间行/列的权重加倍（2 而不是 1），这是为了：

1. **强调中心**：离中心像素越近的邻域像素对梯度贡献应该越大
2. **噪声抑制**：相比于简单的 [-1, 1] 差分，Sobel 提供了更好的噪声平滑效果

#### 梯度幅值

梯度是一个二维向量 **(Gx, Gy)**：

- **幅值（Magnitude）**：G = √(Gx² + Gy²) — 边缘的"强度"
- **方向（Direction）**：θ = atan2(Gy, Gx) — 边缘的法线方向

```
梯度向量的几何意义:

        Gy ↑
           │   • (Gx, Gy)
           │  /
           │ /  G = √(Gx²+Gy²)
           │/ θ
  ────────┼────────→ Gx
           │
           │

  G 越大 = 边缘越明显
  θ 指示了亮度变化最快的方向（垂直于边缘走向）
```

本题只计算幅值，不计算方向。

#### 图像数据

```
     Col 0  Col 1  Col 2  Col 3  Col 4  Col 5  Col 6  Col 7
Row 0:  50     50    200    200     50     50     50     50
Row 1:  50     50    200    200     50     50     50     50

Row 6:  50     50     50     50     50     50     50     50

```

#### 四种边缘类型

| 边缘类型 | 位置              | 描述                                            |
| -------- | ----------------- | ----------------------------------------------- |
| 垂直边缘 | 行 0-2, 列 1→2 处 | 暗 (50) → 亮 (200) 过渡，Gx 有强响应            |
| 垂直边缘 | 行 0-2, 列 4→5 处 | 亮 (200) → 暗 (50) 过渡，Gx 有负响应 (幅值取正) |
| 水平边缘 | 行 2→3 处         | 混合亮度 → 全亮 (200)，Gy 有强响应              |
| 水平边缘 | 行 5→6 处         | 亮 (200) → 暗 (50)，Gy 有强响应                 |
| 对角边缘 | 行 5 列 0 处      | 暗像素 (50) 与周围亮像素 (200) 形成对角过渡     |
| 均匀区域 | 行 6-7            | 全暗 (50)，Gx = Gy = 0，梯度幅值 = 0            |

### 算法步骤

```
输入: 8×8 灰度图像 IMAGE
输出: 原始图像矩阵、梯度幅值矩阵、边缘二值图像

Step 1: 打印原始图像矩阵
    print_matrix("Original Image:", IMAGE)

Step 2: 对每个像素计算梯度幅值
    for i = 0 to ROWS-1:
        for j = 0 to COLS-1:
            if i == 0 or i == ROWS-1 or j == 0 or j == COLS-1:
                grad_mag[i][j] = 0   // 边界像素, 无完整3×3邻域
            else:
                gx = convolve(IMAGE, GX, i, j)
                gy = convolve(IMAGE, GY, i, j)
                grad_mag[i][j] = round(sqrt(gx² + gy²))

Step 3: 阈值二值化
    for i = 0 to ROWS-1:
        for j = 0 to COLS-1:
            edge_binary[i][j] = (grad_mag[i][j] >= 128) ? 255 : 0

Step 4: 打印梯度幅值矩阵
    print_matrix("Gradient Magnitude:", grad_mag)

Step 5: 打印边缘二值图像
    print_matrix("Edge Map (threshold=128):", edge_binary)
```

### 示例：手工计算

#### 示例 1：垂直边缘处 — 像素 (1, 2)

以 (1,2) 为中心的 3×3 邻域：

```
[ 50] [200] [200]
[ 50] [200] [200]
[ 50] [200] [200]
```

**计算 Gx**（用 GX 核逐元素相乘再求和）：

```
(-1)×50  +  0×200 +  1×200  =  -50 +   0 + 200 = 150
(-2)×50  +  0×200 +  2×200  = -100 +   0 + 400 = 300
(-1)×50  +  0×200 +  1×200  =  -50 +   0 + 200 = 150
                                    Gx = 150+300+150 = 600
```

**计算 Gy**（用 GY 核）：

三行完全相同（无垂直方向变化），正负相消 → **Gy = 0**

**梯度幅值**：G = √(600² + 0²) = **600**

**阈值**：600 ≥ 128 → **255**（白色，边缘像素）

#### 示例 2：水平和垂直边缘交汇处 — 像素 (2, 3)

以 (2,3) 为中心的 3×3 邻域：

```
[ 50] [200] [200]
[ 50] [200] [200]
[200] [200] [200]
```

注意：第三行变为 200（跨过了行 2→3 的水平边缘）。

**计算 Gx**：

```
(-1)×50  +  0×200 +  1×200 = 150
(-2)×50  +  0×200 +  2×200 = 300
(-1)×200 +  0×200 +  1×200 =   0
                         Gx = 450
```

**计算 Gy**：

```
(-1)×50 + (-2)×200 + (-1)×200 = -650
  0 ×50 +   0 ×200 +   0 ×200 =    0
  1 ×200 +  2 ×200 +  1 ×200 =  650
                           Gy = 450
```

**梯度幅值**：G = √(450² + 450²) = √405000 ≈ **636**

Gx 和 Gy 都有响应说明此处同时存在垂直和水平方向的亮度变化。

#### 示例 3：均匀区域 — 像素 (6, 3)

以 (6,3) 为中心的 3×3 邻域（全为 50）：

```
[50] [50] [50]
[50] [50] [50]
[50] [50] [50]
```

正负权重完全相消 → Gx = 0, Gy = 0, G = **0**

阈值：0 < 128 → **0**（黑色，非边缘）

### 边缘检测算子对比

| 算子      | 核大小 | 特点                                | Gx 核示例                       |
| --------- | ------ | ----------------------------------- | ------------------------------- |
| Roberts   | 2×2    | 最简单，对噪声敏感                  | [+1,0; 0,-1]                    |
| Prewitt   | 3×3    | 与 Sobel 类似但无中心加权           | [-1,0,1; -1,0,1; -1,0,1]        |
| **Sobel** | 3×3    | 中心加权，有一定平滑，最常用        | [-1,0,1; -2,0,2; -1,0,1]        |
| Scharr    | 3×3    | 更强的中心加权，旋转对称性更好      | [-3,0,3; -10,0,10; -3,0,3]      |
| Laplacian | 3×3    | 二阶导数，检测边缘但不论方向        | [0,1,0; 1,-4,1; 0,1,0]          |
| Canny     | 多步   | 最优检测器 (S/N 比最高), 需多步处理 | 先用 Sobel+ 非极大抑制 + 双阈值 |

Sobel 的优势：

- 简单高效，只需两个 3×3 卷积
- 中心加权提供一定的噪声平滑
- 同时给出梯度幅值和方向
- 是 Canny 边缘检测的第一步

### 阈值选择

阈值 128 是人为设定的。在实际应用中，阈值的选择至关重要：

```
阈值太低 → 噪声被误检为边缘（假阳性）
阈值太高 → 真实边缘被遗漏（假阴性）

阈值 T 的效果:

  G < T  →  0 (黑色, 非边缘)
  G ≥ T  → 255 (白色, 边缘)

  ┌─────────────────────────┐
  │                         │
  │    非边缘    │   边缘    │
  │    (黑色)   │  (白色)   │
  │             │           │
  └─────────────┼───────────┘
                T
```

常用阈值选取方法：

- **固定阈值**：如本题的 128
- **Otsu 方法**：自动选择使类间方差最大的阈值
- **自适应阈值**：不同区域使用不同阈值

本题中梯度幅值有 0, 212, 474, 600, 636, 671 等多个值，阈值 128 能有效分离边缘与非边缘。

### 常见错误

1. **kernel 索引错误**

   ```
   ❌ kernel[i][j]          — 当 i=-1 时数组越界！
   ✅ kernel[i+1][j+1]      — 将 (-1,0,1) 映射到 (0,1,2)
   ```

2. **忘记处理边界像素**

   ```
   ❌ 对所有像素都调用 convolve → 边界像素读取 image[-1][*] 越界
   ✅ 先判断 i==0 || i==ROWS-1 || j==0 || j==COLS-1, 边界设 0
   ```

3. **sqrt 参数类型**

   ```
   ❌ sqrt(gx*gx + gy*gy)         — 整数溢出风险 + 类型警告
   ✅ sqrt((double)(gx*gx + gy*gy)) — 显式转换为 double
   ```

4. **四舍五入方式**

   ```
   ❌ (int)sqrt(...)              — 截断 (truncate), 如 635.6 → 635
   ✅ (int)(sqrt(...) + 0.5)      — 四舍五入，如 635.6 → 636
   ```

5. **忘记链接数学库**

   ```
   ❌ gcc -o sobel sobel.c             — undefined reference to 'sqrt'
   ✅ gcc -o sobel sobel.c -lm         — -lm 链接 libm
   ```

6. **const 限定符冲突**

   ```
   ❌ print_matrix("Title", IMAGE)     — IMAGE 是 const int[][], 参数是 int[][]
   ✅ print_matrix("Title", (int(*)[COLS])IMAGE)  — 强制转换去掉 const
   ```

7. **输出格式不匹配**

   ```
   ❌ printf("%d", mat[i][j])          — 无宽度控制，对齐混乱
   ✅ printf("%3d", mat[i][j])         — 固定 3 位宽度，右对齐
   ```

8. **矩阵后缺少空行**

   ```
   ❌ 打印矩阵后没有额外的 \n
   ✅ 每个矩阵后 printf("\n") 输出空行
   ```

9. **ROWS/COLS 维度搞错**

   ```
   ❌ 仍按 5×5 写循环               — 本题已升级为 8×8
   ✅ 使用 ROWS=8, COLS=8 宏定义     — 循环自动适配
   ```
