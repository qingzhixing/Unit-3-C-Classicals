## Lesson 70 — 量子比特与门电路模拟【量子计算入门】

### 课程任务

用 C 语言实现一个单量子比特（Single Qubit）模拟器，模拟量子态在 Hadamard 门（H）、Pauli-X 门（X）和 Pauli-Z 门（Z）作用下的演化，并最终执行 100 次量子测量。

学员需要完成 6 个函数（对应 6 个 `#error TODO`）：

1. **`complex_mult()`** — 复数乘法：`(a+bi)(c+di) = (ac-bd) + (ad+bc)i`
2. **`complex_add()`** — 复数加法：`(a+bi)+(c+di) = (a+c)+(b+d)i`
3. **`apply_gate()`** — 将 2×2 量子门矩阵作用于量子态向量
4. **`print_state()`** — 以 Dirac 符号格式输出量子态
5. **`measure()`** — 模拟一次投影测量，基于 LCG 伪随机数
6. **`main()`** — 驱动整个模拟流程：初始化 → 四门施加 → 100 次测量

门矩阵常量（`H`、`X`、`Z`）和 `Complex` 结构体已由框架提供，学员只需**使用它们**。

验证命令：

```bash

```

`make test` 编译 qubit.c，运行程序，将输出与 `expected_output.txt` 进行 diff 比较。匹配则通过。

### 前置知识：量子比特（Qubit）

#### 经典比特 vs 量子比特

```
经典比特:                     量子比特:
  ┌───┐                        ┌─────────────────┐
  │ 0 │  或  │ 1 │              │  α|0⟩ + β|1⟩    │
  └───┘      └───┘              │  |α|² + |β|² = 1 │
                                └─────────────────┘
  只能处于两个确定状态之一       可以处于叠加态 (Superposition)
```

量子比特（qubit）是量子计算的基本信息单元。与经典比特不同，量子比特可以同时处于 `|0⟩` 和 `|1⟩` 的**线性叠加**态：

```
|ψ⟩ = α|0⟩ + β|1⟩ = [α]
                     [β]
```

其中 α 和 β 是**复数振幅**（complex amplitudes），满足归一化条件 `|α|² + |β|² = 1`。

#### Dirac 符号（Bra-Ket Notation）

| 符号 | 含义 | 向量表示              |
| ---- | ---- | --------------------- |
| `    | 0⟩`  | 计算基态 0（ket 0）   | `[1, 0]ᵀ`              |
| `    | 1⟩`  | 计算基态 1（ket 1）   | `[0, 1]ᵀ`              |
| `    | ψ⟩`  | 任意量子态（ket psi） | `[α, β]ᵀ`              |
| `⟨ψ  | `    | 对偶向量（bra psi）   | `[α*, β*]`（共轭转置） |
| `⟨φ  | ψ⟩`  | 内积（bra-ket）       | 复数标量               |

#### Bloch 球表示

任何单量子比特的纯态都可以映射到单位球面（Bloch 球）上的一点：

```
                    |0⟩ (北极)
                    ▲
                   /│\
                  / │ \
                 /  │  \
                /   ●   \    ← 任意状态 |ψ⟩
               /    │    \
              /     │     \
             /      │      \
            └───────┼───────┘
                    │
                    ▼
                   |1⟩ (南极)

  |ψ⟩ = cos(θ/2)|0⟩ + e^(iφ) sin(θ/2)|1⟩

  θ ∈ [0, π]    极角 (polar angle) — 决定 |0⟩ 和 |1⟩ 的概率分布
  φ ∈ [0, 2π)   方位角 (azimuthal angle) — 决定相对相位
```

| 状态 | Bloch 球位置 | 向量 | θ         | φ          |
| ---- | ------------ | ---- | --------- | ---------- |
| `    | 0⟩`          | 北极 | `[1, 0]ᵀ` | 0          | 任意             |
| `    | 1⟩`          | 南极 | `[0, 1]ᵀ` | π          | 任意             |
| `    | +⟩ = (       | 0⟩+  | 1⟩)/√2`   | X 轴正方向 | `[1/√2, 1/√2]ᵀ`  | π/2 | 0   |
| `    | -⟩ = (       | 0⟩-  | 1⟩)/√2`   | X 轴负方向 | `[1/√2, -1/√2]ᵀ` | π/2 | π   |
| `    | +i⟩ = (      | 0⟩+i | 1⟩)/√2`   | Y 轴正方向 | `[1/√2, i/√2]ᵀ`  | π/2 | π/2 |

### 前置知识：量子门（Quantum Gates）

量子门是作用在量子比特上的**酉变换**（unitary transformation），用 2×2 复数矩阵表示。酉矩阵满足 `U†U = I`（共轭转置等于逆），保证量子操作可逆且保持归一化。

#### 本题使用的三个量子门

##### 1. Hadamard 门（H）

```
H = 1/√2 [[1,  1],
          [1, -1]]

作用:
  H|0⟩ = (|0⟩ + |1⟩)/√2 = |+⟩    ← 产生叠加态
  H|1⟩ = (|0⟩ - |1⟩)/√2 = |-⟩

性质:
  H² = I (自逆，施加两次回到原态)
  H|+⟩ = |0⟩,  H|-⟩ = |1⟩        ← 将叠加态转回计算基

Bloch 球: 绕 (x+z)/√2 轴旋转 π
```

Hadamard 门是量子计算中最重要的门之一。它将确定的计算基态变为等概率的叠加态，是量子并行性的起点。在 Bloch 球上，它对应绕 XZ 平面对角线旋转 180°。

##### 2. Pauli-X 门（X / NOT 门）

```
X = [[0, 1],
     [1, 0]]

作用:
  X|0⟩ = |1⟩
  X|1⟩ = |0⟩

性质:
  X² = I
  Bloch 球: 绕 X 轴旋转 π (180°)
```

Pauli-X 门等价于经典 NOT 门——它将 `|0⟩` 翻转为 `|1⟩`，反之亦然。但不同于经典 NOT，它可以作用于叠加态，同时翻转两个分量的振幅。在 Bloch 球上，它对应绕 X 轴旋转 180°。

##### 3. Pauli-Z 门（Z / 相位翻转）

```
Z = [[1,  0],
     [0, -1]]

作用:
  Z|0⟩ =  |0⟩
  Z|1⟩ = -|1⟩

性质:
  Z² = I
  Bloch 球: 绕 Z 轴旋转 π (180°)
```

Pauli-Z 门只改变 `|1⟩` 分量的**相位符号**（乘以 -1），而不改变 `|0⟩` 分量。这在 Bloch 球上对应绕 Z 轴旋转 180°。相位翻转不会改变测量概率（因为 `|−β|² = |β|²`），但在干涉效应中至关重要。

#### 三个 Pauli 矩阵总结

| 门        | 矩阵                | Bloch 旋转轴 | 旋转角度 | 经典类比 |
| --------- | ------------------- | ------------ | -------- | -------- |
| X (NOT)   | `[[0,1],[1,0]]`     | X 轴         | π        | NOT 门   |
| Y         | `[[0,-i],[i,0]]`    | Y 轴         | π        | 无       |
| Z (Phase) | `[[1,0],[0,-1]]`    | Z 轴         | π        | 无       |
| H         | `[[1,1],[1,-1]]/√2` | (X+Z)/√2 轴  | π        | 无       |

### 算法深度解析：状态演化追踪

本题模拟的量子电路：

```
|0⟩ ──[H]──[X]──[Z]──[H]── 测量
```

#### 完整的逐步状态演化

**初始状态** `|ψ₀⟩ = |0⟩`：

```
|ψ₀⟩ = 1·|0⟩ + 0·|1⟩ = [1, 0]ᵀ
α₀ = 1 + 0i,  β₀ = 0 + 0i
P(|0⟩) = |α₀|² = 1.0 (100%)
P(|1⟩) = |β₀|² = 0.0 (0%)
```

**Step 1 — H 门（产生叠加态）**：

```
|ψ₁⟩ = H|ψ₀⟩ = 1/√2 [[1,  1],   [1]   = [1/√2]
                      [1, -1]] × [0]     [1/√2]

     = (|0⟩ + |1⟩)/√2 = |+⟩

α₁ = 1/√2 ≈ 0.707107,  β₁ = 1/√2 ≈ 0.707107
P(|0⟩) = (1/√2)² = 0.5 (50%)
P(|1⟩) = (1/√2)² = 0.5 (50%)

Bloch 球: |+⟩ 位于 X 轴正方向，θ=π/2, φ=0
```

H 门将确定的 `|0⟩` 态变为等概率叠加态 `|+⟩`。此时如果测量，有一半概率得到 `|0⟩`，一半概率得到 `|1⟩`。

**Step 2 — X 门（Pauli-X / NOT）**：

```
|ψ₂⟩ = X|ψ₁⟩ = [[0, 1],   [1/√2]   = [1/√2]
                 [1, 0]] × [1/√2]     [1/√2]

     = |+⟩（与 |ψ₁⟩ 相同！）

α₂ = 1/√2 ≈ 0.707107,  β₂ = 1/√2 ≈ 0.707107
P(|0⟩) = 0.5,  P(|1⟩) = 0.5
```

因为 `|ψ₁⟩ = |+⟩` 的两个振幅相等，X 门交换它们后状态不变。这是 `|+⟩` 态的特殊性质：它是 X 门的**本征态**（eigenstate），本征值为 +1。

**Step 3 — Z 门（Pauli-Z / 相位翻转）**：

```
|ψ₃⟩ = Z|ψ₂⟩ = [[1,  0],   [1/√2]   = [ 1/√2]
                 [0, -1]] × [1/√2]     [-1/√2]

     = (|0⟩ - |1⟩)/√2 = |-⟩

α₃ =  1/√2 ≈  0.707107
β₃ = -1/√2 ≈ -0.707107

P(|0⟩) = |1/√2|² = 0.5 (50%)
P(|1⟩) = |-1/√2|² = 0.5 (50%)

Bloch 球: |-⟩ 位于 X 轴负方向，θ=π/2, φ=π
```

Z 门翻转了 `|1⟩` 分量的符号，将 `|+⟩` 变为 `|-⟩`。注意：虽然 β 的符号变了，但测量概率不变（`|-1/√2|² = 1/2`）。

**Step 4 — 第二个 H 门（回到计算基）**：

```
|ψ₄⟩ = H|ψ₃⟩ = 1/√2 [[1,  1],   [ 1/√2]   = 1/2 × [1-1]   = [0]
                      [1, -1]] × [-1/√2]            [1+1]     [1]

     = |1⟩

α₄ = 0 + 0i,  β₄ = 1 + 0i
P(|0⟩) = |0|² = 0.0 (0%)
P(|1⟩) = |1|² = 1.0 (100%)

Bloch 球: |1⟩ 位于南极，θ=π
```

H 门将 `|-⟩` 变回 `|1⟩`。这是因为 `H|-⟩ = H·(H|1⟩) = H²|1⟩ = |1⟩`（利用了 H²=I 的性质）。

#### 演化总结表

| 步骤 | 操作 | 状态名 | α   | β       | P(0)     | P(1) | Bloch 位置 |
| ---- | ---- | ------ | --- | ------- | -------- | ---- | ---------- |
| 0    | 初始 | `      | 0⟩` | 1+0i    | 0+0i     | 100% | 0%         | 北极          |
| 1    | H    | `      | +⟩` | 1/√2+0i | 1/√2+0i  | 50%  | 50%        | X+ 轴         |
| 2    | X    | `      | +⟩` | 1/√2+0i | 1/√2+0i  | 50%  | 50%        | X+ 轴（不变） |
| 3    | Z    | `      | -⟩` | 1/√2+0i | -1/√2+0i | 50%  | 50%        | X- 轴         |
| 4    | H    | `      | 1⟩` | 0+0i    | 1+0i     | 0%   | 100%       | 南极          |

**关键洞察**：H-X-Z-H 序列的净效果是将 `|0⟩` 变为 `|1⟩`。这等价于一个 X 门。验证：`H·Z·X·H = X`（在忽略全局相位的意义下）。

### 量子测量（Measurement）

#### Born 规则

量子测量是**概率性**的。对状态 `|ψ⟩ = α|0⟩ + β|1⟩` 进行测量：

- 以概率 `|α|² = α.real² + α.imag²` 得到结果 `|0⟩`
- 以概率 `|β|² = β.real² + β.imag²` 得到结果 `|1⟩`

测量后，量子态**坍缩**（collapse）到对应的基态。这意味着量子态被不可逆地破坏——后续测量必然得到相同结果。

#### 本题中的测量

本题使用**线性同余生成器**（Linear Congruential Generator, LCG）模拟测量中的随机性：

```c
/* LCG 公式 (glibc 参数) */
*seed = (1103515245u * (*seed) + 12345u) & 0x7FFFFFFFu;
double r = (double)(*seed) / (double)0x80000000u;  /* r ∈ [0, 1) */

/* 判断坍缩结果 */
if (r < prob0) return 0;  /* 坍缩到 |0⟩ */
else           return 1;  /* 坍缩到 |1⟩ */
```

LCG 参数说明：

- `1103515245u`：乘数（glibc `rand()` 使用的经典参数）
- `12345u`：增量
- `& 0x7FFFFFFFu`：取模 2³¹（保证非负）
- `0x80000000u`：即 2³¹，用于归一化到 [0, 1)

由于最终状态 `|ψ₄⟩ = |1⟩`（`α=0, β=1`），`prob0 = 0`，所以 `r < 0` 永远不成立，100 次测量全部返回 `|1⟩`。

#### 测量 vs 期望值

```
                  ┌─────────────────────────────────────┐
                  │   量子态        测量结果              │
                  │   |ψ⟩ = α|0⟩+β|1⟩                   │
                  │                   ↓                  │
                  │   概率 |α|²  →   |0⟩ (坍缩)          │
                  │   概率 |β|²  →   |1⟩ (坍缩)          │
                  │                                     │
                  │   测量是破坏性的！                    │
                  │   测量后状态不再为 |ψ⟩                │
                  └─────────────────────────────────────┘
```

### 复数运算：实现细节

#### Complex 结构体

```c
typedef struct {
    double real;  /* 实部 */
    double imag;  /* 虚部 */
} Complex;
```

#### 复数乘法

```
(a + bi)(c + di) = ac + adi + bci + bdi²
                 = ac + adi + bci - bd          (因为 i² = -1)
                 = (ac - bd) + (ad + bc)i
```

**实现**：

```c
result.real = a.real * b.real - a.imag * b.imag;
result.imag = a.real * b.imag + a.imag * b.real;
```

#### 复数加法

```
(a + bi) + (c + di) = (a + c) + (b + d)i
```

**实现**：

```c
result.real = a.real + b.real;
result.imag = a.imag + b.imag;
```

#### 矩阵 - 向量乘法（apply_gate 核心）

```
|ψ'⟩ = G |ψ⟩

[α']   [G₀₀  G₀₁]   [α]   [G₀₀·α + G₀₁·β]
[β'] = [G₁₀  G₁₁] × [β] = [G₁₀·α + G₁₁·β]

其中每个乘法是 complex_mult，每个加法是 complex_add
```

**关键陷阱**：必须使用临时变量保存 `new0` 和 `new1`，计算完两个分量后再写回 `state[0]` 和 `state[1]`。否则计算 `new1` 时 `state[0]` 已被覆盖，导致错误结果。

### 常见错误

| 错误                                                             | 后果                                       | 正确做法                                                                       |
| ---------------------------------------------------------------- | ------------------------------------------ | ------------------------------------------------------------------------------ |
| `apply_gate` 中不使用临时变量                                    | `state[0]` 被覆盖后 `state[1]` 计算错误    | 先用 `Complex new0`, `Complex new1` 保存，再写回                               |
| `complex_mult` 中 imag 符号算错：`a.real*b.imag - a.imag*b.real` | 复数乘法结果错误，状态演化全错             | 公式是 `(ac-bd) + (ad+bc)i`，imag 是 **加号**：`a.real*b.imag + a.imag*b.real` |
| `measure` 中 `0x80000000` 不带 `u` 后缀                          | signed int 溢出是**未定义行为** (UB)       | 必须写 `0x80000000u`                                                           |
| `measure` 中 seed 初始值不是 `42u`                               | 随机序列不同，测量统计与预期不一致         | 精确使用 `unsigned int seed = 42u`                                             |
| `print_state` 格式字符串不精确                                   | diff 测试失败                              | 格式必须是 `"( %+f %+fi )                                                      | 0⟩ + ( %+f %+fi ) | 1⟩\n"` |
| `main` 中 `counts[2]` 未初始化为 `{0, 0}`                        | 垃圾值导致统计结果错误                     | 声明时初始化：`int counts[2] = {0, 0}`                                         |
| Makefile 忘记 `-lm`                                              | 链接错误：undefined reference to `sqrt` 等 | `LDFLAGS = -lm`                                                                |
| `Complex` 结构体的 `imag` 字段忘记初始化                         | 垃圾值导致状态错误                         | 始终显式设置 `.real` 和 `.imag`                                                |
| 门矩阵索引混乱：`gate[0][0]` vs `gate[0][1]`                     | 错误的矩阵乘法结果                         | 对照公式检查：`new0 = G[0][0]*s[0] + G[0][1]*s[1]`                             |

### 重要知识点

- **酉变换保持归一化**：量子门是酉矩阵（`U†U = I`），保证 `|α|²+|β|²=1` 始终成立。可以验证：每次 `apply_gate` 后，`|α|²+|β|²` 仍为 1（忽略浮点舍入）。

- **H² = I 的验证**：施加 H 两次回到原态。本题中 `H·H = I`，所以 `H·Z·X·H|0⟩ = H·Z·X·H|0⟩`。而 `H·Z·X·H` 的矩阵乘积恰为 X。

- **相位不影响测量概率**：Z 门翻转了 β 的符号（`β→-β`），但 `|-β|² = |β|²`，测量概率不变。相位在干涉效应中才体现——例如 H 门将 `|+⟩` 和 `|-⟩` 映射到不同的计算基态。

- **量子并行性**：n 个量子比特可以同时表示 2ⁿ 个基态的叠加。这是量子计算加速的来源。本题的单量子比特只是入门——真正的量子优势需要多量子比特纠缠。

- **测量是不可逆的**：测量后量子态坍缩到本征态，原来的叠加信息永久丢失。这是量子力学与经典力学的根本区别之一。

- **不可克隆定理**：未知量子态不能被完美复制。这是量子密码学（如 BB84 协议）安全性的理论基础。

- **量子门是线性的**：`U(α|0⟩+β|1⟩) = αU|0⟩ + βU|1⟩`。这意味着知道门对基态的作用，就能推出对任意叠加态的作用。

- **Bloch 球可视化**：所有单量子比特纯态对应 Bloch 球面上的点。量子门对应球的旋转。例如 X 门 = 绕 X 轴旋转 180°，Z 门 = 绕 Z 轴旋转 180°，H 门 = 绕 (X+Z)/√2 轴旋转 180°。

### 课堂讨论

1. **为什么 `apply_gate` 必须使用临时变量？** — 因为矩阵 - 向量乘法的两个分量都依赖于原始的 `state[0]` 和 `state[1]`。如果先计算并写回 `state[0]`，再计算 `state[1]` 时读到的 `state[0]` 已经是新值而非旧值。这是典型的"写后读"数据竞争——即便在单线程程序中也会发生。正确做法是用临时变量保存两个新分量，计算完再一起写回。

2. **如果改变门的作用顺序（如 H→Z→X→H），最终状态是什么？** — 矩阵乘法不交换，顺序至关重要。`H·Z·X·H ≠ H·X·Z·H`。对于 `|0⟩` 初始态：`H·X·Z·H|0⟩ = |0⟩`（回到原态），而 `H·Z·X·H|0⟩ = |1⟩`。这是因为 X 和 Z 不对易（`XZ = -ZX`）。

3. **测量 100 次全部返回 `|1⟩`，为什么不是概率性的？** — 因为最终状态 `|ψ₄⟩ = |1⟩ = [0, 1]ᵀ`，`α=0, β=1`，`prob0 = 0² = 0`。Born 规则给出 P(|0⟩)=0%，P(|1⟩)=100%。这不是"运气好"，而是确定性的。如果在 Step 1 后测量（`|+⟩` 态），才会看到约 50-50 的分布。

4. **为什么本题用 LCG 而不是 `rand()`？** — 为了**可复现性**。`rand()` 的实现在不同平台可能不同，而 LCG 的参数（乘数 1103515245、增量 12345、模 2³¹）是 glibc 的标准参数，保证所有平台产生相同的随机序列。seed=42 是约定俗成的测试种子。这也让 `expected_output.txt` 可以精确预测。

5. **H·X·H 等于什么？** — `H·X·H = Z`。可以手动验证：`H·X·H|0⟩ = H·X·(|0⟩+|1⟩)/√2 = H·(|1⟩+|0⟩)/√2 = H·(|0⟩+|1⟩)/√2 = |0⟩`。而 `Z|0⟩ = |0⟩`。类似地 `H·Z·H = X`。这说明 Hadamard 门可以"旋转"Pauli 矩阵的基底——X 和 Z 通过 H 共轭互换。

6. **如果 `prob0` 非常小但不为零（如 0.000001），测量 100 次一定能看到 `|0⟩` 吗？** — 不一定。概率不保证频率。期望次数是 100 × 0.000001 = 0.0001 次，即大概率 100 次都是 `|1⟩`。但"大概率"不是"一定"。量子力学的概率性是本质的，不是由于信息不完备——这与经典统计力学有根本区别。在本题中，由于 `prob0` 精确为 0（最终态是纯 `|1⟩`），所以确定性地全部返回 `|1⟩`。
