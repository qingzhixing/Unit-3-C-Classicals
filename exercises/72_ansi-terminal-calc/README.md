## Lesson 72 — ANSI 终端计算器：双栈表达式求值 + 终端控制序列

### 课程任务

实现一个终端四则运算计算器，使用**双栈算法**对固定表达式求值，并用 **ANSI 转义序列**美化彩色输出。

**子任务**：

1. `parse_number(s, pos)` — 从字符串位置 `*pos` 提取整数，支持多位数
2. `precedence(op)` — 返回运算符优先级（`*/` 为 2，`+-` 为 1）
3. `apply_op(a, b, op)` — 对两个操作数执行四则运算
4. `evaluate(s)` — 双栈算法核心：操作数栈 + 运算符栈协同求值
5. `print_colored(color, text)` — 用 ANSI 颜色打印文本
6. `main()` — 驱动两个测试表达式并打印彩色结果

**测试表达式**：

| 表达式      | 期望结果 | 考点                     |
| ----------- | -------- | ------------------------ |
| `"3+4*2"`   | 11       | 运算符优先级（先乘后加） |
| `"(5+3)*2"` | 16       | 括号改变优先级           |

**验证方式**：`make test` 将程序输出与 `expected_output.txt` 逐字节对比（包含 ANSI 转义序列）。

### 前置知识

#### 1. 表达式求值的三种经典方法

```
方法一：双栈算法 (Dijkstra)
  操作数栈 nums[]  +  运算符栈 ops[]
  一遍扫描，按优先级弹栈计算
  特点：简单直观，适合教学 ✓

方法二：递归下降解析 (Recursive Descent)
  expr() → term() → factor()
  函数递归调用编码优先级
  特点：结构化好，适合编译器前端

方法三：逆波兰表达式 (RPN / Postfix)
  中缀 → 后缀 → 求值（两遍）
  特点：适合栈式计算机（如 Forth 语言）
```

本题使用**方法一（双栈算法）**，这是最直观的实现方式。

#### 2. 运算符优先级与结合性

```
优先级表（C语言风格）：
┌─────────────────┬──────────┬──────────┐
│ 运算符           │ 优先级    │ 结合性    │
├─────────────────┼──────────┼──────────┤
│ (               │ 0 (特殊)  │ —        │
│ +  -            │ 1 (低)    │ 左结合    │
│ *  /            │ 2 (高)    │ 左结合    │
└─────────────────┴──────────┴──────────┘
```

**左结合**意味着 `a - b - c` 被解析为 `(a - b) - c`，而不是 `a - (b - c)`。
双栈算法天然支持左结合：遇到同优先级运算符时弹栈计算（`>=` 条件）。

#### 3. 双栈数据结构

```
操作数栈 nums[]                 运算符栈 ops[]
┌───┬───┬───┬───┬───┐          ┌───┬───┬───┬───┬───┐
│ 3 │ 4 │ 2 │   │   │          │ + │ * │   │   │   │
└───┴───┴───┴───┴───┘          └───┴───┴───┴───┴───┘
  ↑ ntop = 3                     ↑ otop = 2

压栈: nums[ntop++] = value       ops[otop++] = ch
弹栈: value = nums[--ntop]       ch = ops[--otop]
```

栈是用数组模拟的：`ntop` 和 `otop` 始终指向**下一个空位**。
弹栈时先 `--` 再取值，压栈时先存值再 `++`。

#### 4. ANSI 转义序列基础

```
ANSI 转义序列结构:
  ESC [ 参数 m

  ESC = \033 (八进制) = 0x1B (十六进制) = ^[ (控制字符)

  例如: \033[32m  =  设置前景色为绿色

颜色代码速查:
┌──────────────┬──────────┬──────────────────┐
│ 序列          │ 效果      │ 本题中的宏        │
├──────────────┼──────────┼──────────────────┤
│ \033[31m     │ 红色前景  │ COLOR_RED        │
│ \033[32m     │ 绿色前景  │ COLOR_GREEN      │
│ \033[33m     │ 黄色前景  │ COLOR_YELLOW     │
│ \033[36m     │ 青色前景  │ COLOR_CYAN       │
│ \033[1m      │ 加粗/高亮 │ COLOR_BOLD       │
│ \033[0m      │ 重置所有  │ COLOR_RESET      │
└──────────────┴──────────┴──────────────────┘

使用模式:
  printf(COLOR_GREEN "正确!" COLOR_RESET "\n");
         ↑           ↑      ↑
         开启颜色    文本    恢复默认
```

**关键规则**：每次颜色输出后必须用 `COLOR_RESET` 恢复，否则颜色会"泄漏"到后续所有输出。

### 算法详解

#### 双栈求值核心逻辑

```
算法伪代码:

function evaluate(s):
    nums[64], ops[64]
    ntop = 0, otop = 0
    i = 0

    while i < strlen(s):
        if s[i] is space:      i++, continue
        if s[i] is digit:      val = parse_number(s, &i)
                                nums[ntop++] = val
        if s[i] is '(':        ops[otop++] = '('
                                i++
        if s[i] is ')':        while ops[otop-1] != '(':
                                    b = nums[--ntop]
                                    a = nums[--ntop]
                                    op = ops[--otop]
                                    nums[ntop++] = apply_op(a, b, op)
                                otop--  // pop '('
                                i++
        if s[i] is operator:   while otop>0 AND ops[otop-1]!='('
                                    AND precedence(ops[otop-1]) >= precedence(s[i]):
                                    b = nums[--ntop]
                                    a = nums[--ntop]
                                    op = ops[--otop]
                                    nums[ntop++] = apply_op(a, b, op)
                                ops[otop++] = s[i]
                                i++

    while otop > 0:             // 处理剩余运算符
        b = nums[--ntop]
        a = nums[--ntop]
        op = ops[--otop]
        nums[ntop++] = apply_op(a, b, op)

    return nums[0]
```

#### 关键设计决策

**为什么弹栈条件是 `>=` 而不是 `>`？**

```
表达式: 3 - 2 - 1

如果用 > (仅弹更高优先级):
  nums: [3, 2, 1]  ops: [-, -]
  最后弹栈: 先算 2-1=1, 再算 3-1=2  ← 错误! (正确是 0)

如果用 >= (弹同优先级或更高):
  读到第二个 '-' 时, 栈顶 '-' 优先级 >= 当前 '-'
  先弹栈: 3-2=1, nums: [1], ops: [-]
  再压入 '-', nums: [1, 1], ops: [-]
  最后: 1-1=0  ← 正确! ✓
```

`>=` 保证了**左结合性**。

**为什么 `(` 的优先级是 0？**

`(` 在栈内时不应被弹出，只有当遇到 `)` 时才显式处理。将其优先级设为 0 确保 `precedence('(') >= prec` 对任何 `prec >= 1` 都不成立，从而 `(` 不会被弹栈条件触发。

### 完整逐步跟踪

#### 跟踪 1：`"3+4*2"` → 11

```
步骤  读取  操作                              nums         ops         输出
────  ────  ────────────────────────────────  ───────────  ───────────  ──────────────────────────
初始化                                        [ ]          [ ]
  1    '3'  数字, parse_number → 3            [3]          [ ]          parse_number: 3
  2    '+'  运算符, prec=1, 栈空直接压栈       [3]          [+]          operator: + (precedence=1)
  3    '4'  数字, parse_number → 4            [3,4]        [+]          parse_number: 4
  4    '*'  运算符, prec=2                     [3,4]        [+,*]        operator: * (precedence=2)
            栈顶'+'(prec=1) < 当前(2), 不弹
  5    '2'  数字, parse_number → 2            [3,4,2]      [+,*]        parse_number: 2
  6    END  弹栈: 弹出'*' → 4*2=8             [3,8]        [+]          apply_op: 4 * 2 = 8
  7    END  弹栈: 弹出'+' → 3+8=11            [11]         [ ]          apply_op: 3 + 8 = 11
────  ────  ────────────────────────────────  ───────────  ───────────  ──────────────────────────
结果: 11 ✓
```

#### 跟踪 2：`"(5+3)*2"` → 16

```
步骤  读取  操作                              nums         ops         输出
────  ────  ────────────────────────────────  ───────────  ───────────  ──────────────────────────
初始化                                        [ ]          [ ]
  1    '('  左括号, 直接压栈                   [ ]          [(]          enter paren: (
  2    '5'  数字, parse_number → 5            [5]          [(]          parse_number: 5
  3    '+'  运算符, prec=1                     [5]          [(,+]        operator: + (precedence=1)
            栈顶'(', 不弹栈 (括号是边界)
  4    '3'  数字, parse_number → 3            [5,3]        [(,+]        parse_number: 3
  5    ')'  右括号                             [5,3]        [(,+]        exit paren: )
            弹栈直到 '(': 弹出'+' → 5+3=8     [8]          [(]
            弹出 '('                          [8]          [ ]
  6    '*'  运算符, prec=2, 栈空直接压栈       [8]          [*]          operator: * (precedence=2)
  7    '2'  数字, parse_number → 2            [8,2]        [*]          parse_number: 2
  8    END  弹栈: 弹出'*' → 8*2=16            [16]         [ ]          apply_op: 8 * 2 = 16
────  ────  ────────────────────────────────  ───────────  ───────────  ──────────────────────────
结果: 16 ✓
```

### 边界情况分析

| 场景                 | 预期行为   | 原因                           |
| -------------------- | ---------- | ------------------------------ |
| 单数字 `"42"`        | 返回 42    | 无运算符时直接返回 nums[0]     |
| 多余空格 `" 3 + 4 "` | 返回 7     | while 循环跳过所有空格         |
| 嵌套括号 `"((1+2))"` | 返回 3     | 递归弹栈正确处理多层括号       |
| 连续运算符 `"3+-2"`  | 未定义     | 本题不测试此类表达式           |
| 除零 `"5/0"`         | 返回 0     | apply_op 检查 b!=0             |
| 空字符串 `""`        | 返回 0     | nums 栈为空，nums[0] 未初始化  |
| 多位数 `"123+456"`   | 返回 579   | parse_number 的 while 循环处理 |
| 不匹配括号 `"(1+2"`  | 结果不确定 | 本题不测试此类输入             |

### 常见错误

| 错误                                    | 后果                                        | 正确做法                                 |
| --------------------------------------- | ------------------------------------------- | ---------------------------------------- |
| `isdigit(s[*pos])` 不转 `unsigned char` | 中文字符/负值 char 导致未定义行为           | `isdigit((unsigned char)s[*pos])`        |
| `parse_number` 用 `if` 而非 `while`     | 只读取一位数字，"42" 被解析为 4             | 使用 `while` 循环累积 digit              |
| 弹栈计算时先取 a 再取 b                 | `a-b` 变成 `b-a`, 结果符号反转              | 先 `b=nums[--ntop]`, 再 `a=nums[--ntop]` |
| 弹栈条件用 `>` 而非 `>=`                | 左结合运算符变成右结合，`3-2-1` 得 2 而非 0 | 使用 `>=` 保证左结合                     |
| 忘记在 `)` 处理中弹出 `(`               | ops 栈残留 `(`, 后续计算错乱                | `if (otop > 0) otop--;`                  |
| 遍历结束后不处理剩余 ops                | 栈中运算符未执行，结果错误                  | `while (otop > 0)` 弹栈到底              |
| ANSI 颜色后不加 `COLOR_RESET`           | 后续输出全变成该颜色                        | 每次颜色输出后立即重置                   |
| `precedence` 中 `(` 返回非零值          | `(` 被错误弹出，括号失效                    | `(` 返回 0                               |
| 输出多余空格或换行                      | `diff` 失败                                 | 严格匹配 `expected_output.txt` 格式      |
| `COLOR_BOLD` 和颜色组合写错顺序         | 样式不生效                                  | `"\033[1m\033[33m"` 先样式后颜色         |

### 重要知识点

1. **双栈算法是 Dijkstra 在 1960 年代提出的**，最初用于 ALGOL 60 编译器。它的优雅之处在于只需一遍扫描，无需构建显式语法树。

2. **`>=` 条件是左结合性的关键**。如果使用 `>`，`a-b-c` 会被错误计算为 `a-(b-c)`。这是双栈算法最常见的实现错误。

3. **ANSI 转义序列是带内信令**（in-band signaling）：控制信息与数据在同一字节流中传输。这意味着 `diff` 能直接比较 ANSI 序列，无需特殊处理。

4. **`isdigit` 的参数必须是 `unsigned char` 或 `EOF`**。C 标准规定，如果 `char` 为有符号且值为负（如中文 UTF-8 字节），传入 `isdigit` 是未定义行为。

5. **栈用数组模拟而非链表**：本题表达式长度有限（< 64 个 token），数组栈更简单高效。在生产代码中，长表达式需要动态扩容。

6. **括号通过将 `(` 压入 ops 栈处理**。`(` 在栈内作为"屏障"：遇到 `)` 时弹栈直到 `(`，自然支持嵌套。这是双栈算法的精髓。

7. **`COLOR_RESET` 不仅是好习惯，更是必须**。没有它，终端状态被污染，后续输出（包括 shell 提示符）都会变色。这在管道和重定向场景中同样重要。

8. **整数除法的截断行为**：C 语言中 `a/b` 向零截断（C99+），`15/3=5` 恰好整除。如果扩展支持 `7/3`，结果是 `2` 而非 `2.333`。

### 课堂讨论

1. **如果要把计算器扩展为支持浮点数（如 `3.14*2`），需要修改哪些部分？**

   `parse_number` 需要支持小数点读取，`nums` 栈需要改为 `double` 类型，`apply_op` 的除法不再需要除零检查（浮点除零得 `inf`）。输出格式也需要改为 `%g` 或 `%f`。核心的双栈逻辑本身不变。

2. **为什么本题选择双栈算法而不是递归下降？两种方法各有什么优劣？**

   双栈算法更直观——栈的压入弹出直接对应计算过程，学生容易在纸上跟踪。递归下降需要理解词法分析器 + 递归函数调用，概念负担更重。但递归下降更容易扩展（加幂运算 `**` 只需加一个函数），也更接近真实编译器。

3. **ANSI 转义序列在 `diff` 比较时会产生什么问题？如何调试？**

   ANSI 序列是不可见字符，肉眼无法在 diff 输出中看到差异。调试方法：(a) `cat -v` 显示 `^[`；(b) `xxd` 查看十六进制；(c) 先输出纯文本版本确认逻辑正确，再加颜色。

4. **`precedence('(')` 为什么返回 0？如果返回 3 会怎样？**

   返回 0 确保 `(` 不会被任何运算符的 `>=` 条件触发弹栈。如果返回 3，当遇到 `+`（优先级 1）时，`precedence('(') >= 1` 为真，`(` 会被错误弹出并尝试作为运算符执行 `apply_op`，导致混乱。

5. **如果表达式包含非法字符（如 `3&4`），当前实现会怎样？如何改进？**

   当前实现中 `&` 不匹配任何已知运算符，会被跳过（最后的 `i++`）。结果不确定。改进：在 default 分支打印错误信息并返回错误码，或在 `evaluate` 中增加错误 token 检测。

6. **`make test` 使用 `| diff - expected_output.txt` 进行验证。如果学生在 Windows 上开发（`\r\n` 换行），会遇到什么问题？**

   Windows 的 `\r\n` 会导致输出多一个 `\r` 字节，diff 失败。解决：使用 `-std=c99` 编译时 C 运行时会处理，或者使用 `dos2unix` 转换。本题在 Linux 环境验证，不涉及此问题。
