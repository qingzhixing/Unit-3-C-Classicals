# Lesson 59 — NFA 模拟与子集构造→DFA【自动机理论实战】

### 课程任务

实现 NFA（非确定有限自动机）的三个核心算法，完成从 NFA 到 DFA 的完整转换流程：

1. **`e_closure()`** — ε-闭包计算：给定状态集（位掩码），返回所有经ε转移可达的状态
2. **`NFA_simulate()`** — NFA 模拟：给定输入串，逐步跟踪状态集变化，判断是否被 NFA 接受
3. **`subset_construct()`** — 子集构造算法：将 NFA 转化为等价的 DFA，打印 DFA 转换表

固定 NFA 识别语言 **`a*b | ab*`**（零个或多个 a 后接一个 b，或者一个 a 后接零个或多个 b）：

```
状态数: 5 (状态0~4)
字母表: {a, b}
ε转移:  0→1, 0→2
a转移:  1→1, 2→4
b转移:  1→3, 4→4
初态:   0
接受态: {3, 4}   (两分支各自独立的接受态)
```

测试输入串 `"aab"`（应接受）和 `"aba"`（应拒绝），输出ε-闭包结果、NFA 模拟每步状态集、DFA 转换表。

### 前置知识：自动机理论基础

#### 什么是有限自动机（Finite Automaton）？

有限自动机是计算理论中最基础的数学模型，用于描述和识别正则语言。一个 FA 由五个要素定义：

```
M = (Q, Σ, δ, q₀, F)

Q  : 有限状态集       → 本题: Q = {0, 1, 2, 3, 4}
Σ  : 有限输入字母表   → 本题: Σ = {a, b}
δ  : 转移函数         → 本题: 见转移表
q₀ : 初态            → 本题: q₀ = 0
F  : 接受态集         → 本题: F = {3, 4}
```

#### DFA vs NFA：确定性 vs 非确定性

```
        DFA                          NFA
        ───                          ───
  ┌─a─→ 1                  ┌─a─→ 1 ←─a─┐
  0         每个状态对      0──ε─→ 2    可以有ε转移
  └─b─→ 2   每个符号只有    └─a─→ 3    同一符号可有多条转移
            一条转移
```

| 维度         | DFA                            | NFA                                    |
| ------------ | ------------------------------ | -------------------------------------- |
| 转移确定性   | 每个状态对每个符号**唯一**后继 | 可有多条转移（包括 0 条）              |
| ε转移        | **不允许**                     | **允许**（空串转移）                   |
| 状态数       | 可能很多（最坏 2ⁿ）            | 通常较少                               |
| 模拟复杂度   | O(n) — 每步只需查一个状态      | O(n·\|Q\|) — 需维护状态**集合**        |
| 能否判定接受 | 简单：读完串看当前状态         | 需维护状态集，读完看是否**包含**接受态 |
| 表达能力     | 正则语言                       | **等价！**（正则语言）                 |

**核心定理（Rabin-Scott, 1959）**：对任意 NFA，存在等价的 DFA 识别相同语言。子集构造算法就是证明这个定理的构造性方法。

#### 本题 NFA 的完整状态转移图

```
                    ┌──────────┐
                    │   a      │  (a* 自环)
                    └──→ 1 ───┘
                   ↗ ε       │ b
                  /          ↓
          ┌─────/           (3)  ← 接受态① (无出边 ⇒ a*b 恰好一个 b)
          │  0
          └─────\      a
                  ↘ ε   \
                    └──→ 2 ──a──→ (4)  ← 接受态②
                                  │
                                  └── b ──→ 4  (b* 自环)

  分支①(经 1): a* 后一个 b → 接受态 3      = a*b
  分支②(经 2): 一个 a → 接受态 4，再 b* 自环 = ab*
  两分支【接受态不同】(3 与 4)，故 a*b 分支不会搭上 b 自环而误收 a*b⁺。
```

**语言 `a*b | ab*` 的含义**：

- `a*b`：零个或多个 `a`，后接恰好一个 `b`。例如：`b`、`ab`、`aab`、`aaab`
- `ab*`：恰好一个 `a`，后接零个或多个 `b`。例如：`a`、`ab`、`abb`、`abbb`
- 两者的并集。注意：`ab` 同时属于两个分支。

### 算法详解

#### 算法 1：ε-闭包（ε-closure）

**定义**：ε-closure(S) = 从集合 S 中任意状态出发，仅经过ε转移（零步或多步）能到达的所有状态的集合。

**为什么需要ε-闭包？** 因为 NFA 在任何时刻都可以"免费"地沿ε转移移动，不需要消耗任何输入符号。因此，NFA 在某时刻的"真实"可能状态集，必须包含所有ε可达的状态。

**不动点迭代算法**：

```
输入: 状态集 S（位掩码）
输出: ε-closure(S)

closure = S
repeat:
    changed = false
    for each state s in closure:
        for each ε-target t of s:
            if t not in closure:
                closure = closure ∪ {t}
                changed = true
until not changed
return closure
```

**本题ε-闭包计算示例**：

```
e-closure({0}) 的计算过程:
──────────────────────────────────────
初始: closure = {0}                    (0的位掩码: 0001 = 1)

第1轮扫描:
  状态0在closure中, ε目标: 1, 2
    → 1不在closure中 → 加入, closure = {0,1}
    → 2不在closure中 → 加入, closure = {0,1,2}
  changed = true

第2轮扫描:
  状态0: ε目标1,2 已在closure中
  状态1: ε目标: 无
  状态2: ε目标: 无
  changed = false → 不动点!

结果: e-closure({0}) = {0,1,2}
```

```
各状态的ε-闭包:
  e-closure({0}) = {0, 1, 2}   ← 0通过ε可达1和2
  e-closure({1}) = {1}         ← 1没有ε转移
  e-closure({2}) = {2}         ← 2没有ε转移
  e-closure({3}) = {3}         ← 3没有ε转移
```

#### 算法 2：NFA 模拟

**核心思想**：NFA 运行时，不是处于"某一个"状态，而是处于"一组可能的状态"（状态集）。每读入一个符号，计算新的状态集。

```
输入: 字符串 w = w₁w₂...wₙ
输出: ACCEPT 或 REJECT

current = e-closure({q₀})          ← 初始状态集（含ε闭包）

for i = 1 to n:
    next = ∅
    for each state s in current:   ← 对当前每个可能状态
        for each t in δ(s, wᵢ):    ← 查symbol转移
            next = next ∪ {t}
    current = e-closure(next)       ← 再求ε闭包

if current ∩ F ≠ ∅:                ← 若状态集中包含接受态
    return ACCEPT
else:
    return REJECT
```

**关键细节**：每步分为两个阶段——先做 symbol 转移（消耗一个输入字符），再做ε-闭包（"免费"移动）。顺序不能颠倒！

**输入串 "aab" 的 NFA 模拟跟踪**：

```
Step 0 (初始):
  e-closure({0}) = {0, 1, 2}
  状态集: {0,1,2}

Step 1 (读 'a'):
  symbol转移: 从{0,1,2}出发
    0 --a--> 无转移
    1 --a--> {1}       (1的自环)
    2 --a--> {4}       (2的唯一a转移)
  → 到达: {1, 4}
  e-closure({1,4}) = {1, 4}        (1和4都没有ε转移)
  状态集: {1,4}

Step 2 (读 'a'):
  symbol转移: 从{1,4}出发
    1 --a--> {1}
    4 --a--> 无转移
  → 到达: {1}
  e-closure({1}) = {1}
  状态集: {1}

Step 3 (读 'b'):
  symbol转移: 从{1}出发
    1 --b--> {3}
  → 到达: {3}
  e-closure({3}) = {3}
  状态集: {3}

最终: {3} ∩ {3,4} ≠ ∅ → ACCEPT ✓  (aab = a*b, 属于该语言)
```

**输入串 "aba" 的 NFA 模拟跟踪**：

```
Step 0 (初始):
  e-closure({0}) = {0, 1, 2}

Step 1 (读 'a'):
  symbol转移: 1→{1}, 2→{4} → 到达{1,4}
  e-closure({1,4}) = {1, 4}

Step 2 (读 'b'):
  symbol转移: 1→{3}, 4→{4} → 到达{3,4}
  e-closure({3,4}) = {3, 4}

Step 3 (读 'a'):
  symbol转移: 3 --a--> 无, 4 --a--> 无 → 到达∅
  e-closure(∅) = ∅

最终: ∅ ∩ {3,4} = ∅ → REJECT ✓
```

> **为什么用两个接受态？** 若像"朴素"设计那样让两个分支共用一个带 `b` 自环的接受态，`a*b` 分支读完那个 `b` 后还能顺着自环继续吃 `b`，于是 `a*b` 被放大成 `a*b⁺`——`"aabb"`、`"bb"` 会被错误接受。本题用 Thompson 式并联、两分支各自独立的接受态（3 与 4）来杜绝这一经典陷阱。

#### 算法 3：子集构造（Subset Construction）

**核心原理**：NFA 有 n 个状态，DFA 的每个状态对应 NFA 状态集的一个子集。由于幂集大小为 2ⁿ，DFA 最多有 2ⁿ个状态。算法系统地枚举所有"可达"的子集。

```
输入: NFA (Q, Σ, δ, q₀, F)
输出: 等价的DFA (Q', Σ, δ', q₀', F')

1. q₀' = e-closure({q₀})          ← DFA初态 = NFA初态的ε闭包
2. Q' = {q₀'}                      ← 已发现的DFA状态集
3. 未处理队列 = [q₀']

4. while 未处理队列非空:
     S = 取出一个未处理子集

     if S ∩ F ≠ ∅:                 ← S包含NFA接受态
         标记S为DFA接受态

     for each symbol a in Σ:
         T = ∅
         for each state s in S:
             for each t in δ(s, a):
                 T = T ∪ {t}
         T = e-closure(T)           ← 关键: 先symbol转移再ε闭包

         if T ≠ ∅:
             if T ∉ Q':             ← 新子集
                 Q' = Q' ∪ {T}
                 将T加入未处理队列
             δ'(S, a) = T
         else:
             δ'(S, a) = 死状态(或留空)

5. 输出 Q', δ', q₀', F'
```

**本题 NFA→DFA 转换全过程**：

```
初始:
  DFA初态 D0 = e-closure({0}) = {0,1,2}
  Q' = {D0}, 未处理 = [D0]
  接受态集 F = {3, 4}

────────────────────────────────────────────
处理 D0 = {0,1,2}:  接受态? {3,4}∩{0,1,2}=∅ → No
  'a': 0→无, 1→{1}, 2→{4}  →  {1,4}; e-closure={1,4}
       → 新子集! D1 = {1,4};  δ'(D0,a)=D1
  'b': 0→无, 1→{3}, 2→无    →  {3}; e-closure={3}
       → 新子集! D2 = {3};   δ'(D0,b)=D2

────────────────────────────────────────────
处理 D1 = {1,4}:  接受态? 4∈{1,4} → Yes!
  'a': 1→{1}, 4→无          →  {1}; e-closure={1}
       → 新子集! D3 = {1};   δ'(D1,a)=D3
  'b': 1→{3}, 4→{4}         →  {3,4}; e-closure={3,4}
       → 新子集! D4 = {3,4}; δ'(D1,b)=D4

────────────────────────────────────────────
处理 D2 = {3}:  接受态? 3∈{3} → Yes!  (状态3无出边)
  'a': 3→无 → ∅ → 死状态(无转移)
  'b': 3→无 → ∅ → 死状态(无转移)

────────────────────────────────────────────
处理 D3 = {1}:  接受态? {3,4}∩{1}=∅ → No
  'a': 1→{1} → {1}=D3 (自环);  δ'(D3,a)=D3
  'b': 1→{3} → {3}=D2;         δ'(D3,b)=D2

────────────────────────────────────────────
处理 D4 = {3,4}:  接受态? {3,4}∩{3,4}≠∅ → Yes!
  'a': 3→无, 4→无 → ∅ → 死状态
  'b': 3→无, 4→{4} → {4}; e-closure={4}
       → 新子集! D5 = {4};  δ'(D4,b)=D5

────────────────────────────────────────────
处理 D5 = {4}:  接受态? 4∈{4} → Yes!
  'a': 4→无 → ∅ → 死状态
  'b': 4→{4} → {4}=D5 (自环);  δ'(D5,b)=D5

────────────────────────────────────────────
所有DFA状态处理完毕! 共6个DFA状态。
```

**最终 DFA 转换表**：

```
┌──────────┬──────────────┬────────┬────────┬─────────┐
│ DFA状态  │  NFA子集     │   a    │   b    │ 接受态? │
├──────────┼──────────────┼────────┼────────┼─────────┤
│ D0       │ {0, 1, 2}    │  D1    │  D2    │ No      │
│ D1       │ {1, 4}       │  D3    │  D4    │ Yes     │
│ D2       │ {3}          │  —     │  —     │ Yes     │
│ D3       │ {1}          │  D3    │  D2    │ No      │
│ D4       │ {3, 4}       │  —     │  D5    │ Yes     │
│ D5       │ {4}          │  —     │  D5    │ Yes     │
└──────────┴──────────────┴────────┴────────┴─────────┘
```

**DFA 状态转移图**（接受态用双圈 (( )) 表示）：

```
  a*b 分支:            ab* 分支:
       a (自环)                    b (自环)
      ┌───┐                       ┌───┐
      ▼   │                       ▼   │
  D0 ─a─▶ D3 ─b─▶ ((D2))     D1 ─b─▶ ((D4)) ─b─▶ ((D5))
   │       ▲(D3 a 自环)         ▲              (D5 b 自环)
   │ b     │                    │ a
   ▼       │                    │
 ((D2))    └──── D0 ─a─▶ D1 ────┘

    初态: D0        接受态: D1, D2, D4, D5
    D2 = {3} 读完唯一的 b 即接受, 之后任何字符都进死状态(a*b 恰好一个 b)
```

### 位掩码状态集表示

本题使用 `int` 的低 5 位表示 NFA 状态集，这是一种高效且优雅的编码方式：

```
位掩码编码:
  状态0 → bit 0 (值 1  = 0x01)
  状态1 → bit 1 (值 2  = 0x02)
  状态2 → bit 2 (值 4  = 0x04)
  状态3 → bit 3 (值 8  = 0x08)
  状态4 → bit 4 (值 16 = 0x10)

常用操作:
  加入状态s:       states |= (1 << s)
  检查状态s:       if (states & (1 << s))
  空集判断:        if (states == 0)
  包含任一接受态:  if (states & ACCEPT_MASK)   /* ACCEPT_MASK = (1<<3)|(1<<4) */
```

| 状态集  | 位掩码值 | 二进制 |
| ------- | -------- | ------ |
| ∅       | 0        | 00000  |
| {0}     | 1        | 00001  |
| {1}     | 2        | 00010  |
| {3}     | 8        | 01000  |
| {4}     | 16       | 10000  |
| {0,1,2} | 7        | 00111  |
| {1,4}   | 18       | 10010  |
| {3,4}   | 24       | 11000  |

### DFA 最小化概念

子集构造产生的 DFA 不一定是最小的。本题子集构造得到 6 个 DFA 状态，其中 **D4 和 D5 是等价的、可以合并**——这正好是"子集构造 ≠ 最小 DFA"的一个真实例子。

**DFA 最小化的核心思想**：将"不可区分"的状态合并。两个状态 p 和 q 是等价的，当且仅当对所有输入串 w，从 p 出发接受 w ⇔ 从 q 出发接受 w。

**Hopcroft 算法**（简要）：

1. 初始划分：接受态组 和 非接受态组
2. 反复细化：若同一组中的状态对某个符号转移到不同组，则分裂
3. 直到不再分裂为止

**本题 DFA 最小化分析**：

```
初始划分(引入死状态 ⊥ 补全所有缺失转移):
  Π₀ = { {D1, D2, D4, D5},   ← 接受态组
         {D0, D3},           ← 非接受态组
         {⊥} }               ← 死状态组

细化接受态组 {D1,D2,D4,D5}:
  看 b: D1→D4(接受组), D2→⊥, D4→D5(接受组), D5→D5(接受组)
     → D2 的 b 落到死状态组, 与其余不同 → 分出 {D2}
  看 a: D1→D3(非接受组), 而 D4/D5→⊥ → 再分出 {D1}
  剩下 {D4, D5}: a 都→⊥, b 都→接受组内(D5) → 不可区分 → 合并!

最终划分: {D0}, {D1}, {D2}, {D3}, {D4≡D5}
结论: 最小 DFA 有 5 个状态; 子集构造的 6 状态里 D4、D5 是冗余的等价对。
```

直观上 D4={3,4} 与 D5={4} 的"未来行为"完全一样：都已接受，读 `a` 进死状态、读 `b` 回到同一个接受态，右语言都是 `b*`，所以可以合并。这说明**子集构造保证正确但不保证最小**，最小化(如 Hopcroft)是其后续步骤。

### 常见错误

| 错误                                                     | 后果                                                   | 正确做法                                           |
| -------------------------------------------------------- | ------------------------------------------------------ | -------------------------------------------------- |
| 位掩码操作写成 `states \|= s` 而非 `states \|= (1 << s)` | 状态集编码完全错误，s=3 时只加了 bit 1 和 bit 0        | 使用 `(1 << s)` 将状态号转为位掩码                 |
| ε-闭包只做一轮扫描（不用 do-while）                      | 若存在ε链（如 0→1→2），只拿到直接ε目标                 | 用 do-while 反复扫描直到不动点                     |
| NFA 模拟中先求ε-闭包再做 symbol 转移                     | 转移结果错误（多走了不该走的ε步）                      | **先 symbol 转移，再ε-闭包**                       |
| 子集构造中忘记对新子集求ε-闭包                           | 生成的 DFA 不等价，可能拒绝应接受的串                  | move(S,a) 之后必须 e_closure                       |
| 检查接受态时写成 `states == (1 << 3)`                    | 若状态集含接受态和其它状态（如{1,4}），或含另一个接受态 4，都会误判为不接受 | 用 `states & ACCEPT_MASK` 检查是否与接受态集相交    |
| 子集构造中 next_subset==0 时也创建新状态                 | 产生多余的"空集状态"                                   | 空集直接标记为无转移                               |
| 数组 `NFA_TRANS[4][2][4]` 第三维未正确以 NO_TARGET 终止  | 循环遍历时越界访问                                     | 确保每个转移列表以 NO_TARGET(-2) 结尾              |
| 混淆 EPSILON_TRANS 和 NFA_TRANS 的终止标记               | ε转移列表被当成 symbol 转移处理                        | EPSILON_TRANS 以 NO_TARGET 终止，与 NFA_TRANS 一致 |

### 重要知识点

- **ε-闭包是不动点计算**：不是一次扫描，而是反复迭代直到没有新状态加入。这是典型的"闭包"概念在计算机中的实现。
- **NFA 模拟 = 状态集的演算**：每一步跟踪的不是"一个状态"而是"一组可能的状态"。这反映了非确定性的本质——NFA 同时探索所有可能路径。
- **子集构造 = 幂集的可达部分**：NFA 有 5 个状态，理论上有 32 个可能子集，但本题只有 6 个是可达的。这体现了"惰性构造"——只生成实际需要的 DFA 状态。
- **先 symbol 转移，再ε闭包**：这个顺序反映了ε转移的语义——ε转移是"免费的"，不消耗输入。如果先做ε闭包再做 symbol 转移，相当于在 symbol 转移前做了额外的ε移动。
- **DFA 不一定比 NFA 大很多**：虽然最坏情况是 2ⁿ膨胀，但实践中许多 NFA 的 DFA 只有多项式大小。本题 4→4 就是例子。
- **位掩码是状态集的紧凑表示**：用 int 的每个 bit 代表一个状态，集合运算变为位运算（并=OR，交=AND，成员测试=位测试），效率极高。
- **正则语言对并/交/补封闭**：`a*b | ab*` 是两个正则语言的并。NFA 的构造直接反映了这种结构化组合。
- **子集构造产生的 DFA 可能有冗余**：本题子集构造得到 6 个状态，其中 D4≡D5 可合并，最小 DFA 只需 5 个状态——正好演示了"子集构造保证正确但不保证最小"，需再经最小化算法（如 Hopcroft）优化。

### 课堂讨论

1. **为什么 NFA 需要ε转移？它有什么实际用途？** — ε转移让 NFA 的构造变得模块化。例如，两个 NFA 的"并"可以通过一个ε转移的新初态连接到两个子 NFA 的初态来实现（Thompson 构造法）。没有ε转移，构造过程会复杂得多。在本题中，ε转移优雅地表达了"要么走 a*b 分支，要么走 ab*分支"的选择。

2. **如果 NFA 的ε-闭包包含回路会怎样？** — 不动点算法仍然正确。例如，若状态 0→ε→1→ε→0 形成ε环，算法会检测到所有ε连通的状态最终都在闭包中。ε环意味着这些状态在ε意义下等价，在最小化时可能会被合并。

3. **子集构造的最坏情况是什么？能举一个例子吗？** — 最坏情况是 2ⁿ个 DFA 状态。经典例子是语言"倒数第 k 个字符是 a"的 NFA，它只需要 k+1 个状态，但任何 DFA 至少需要 2^k 个状态。这说明非确定性在某些情况下可以带来指数级的状态压缩。

4. **本题为什么选择 `a*b | ab*` 这个语言？** — 因为它既有ε转移（两个分支的选择），又有自环（a*和 b*），还有非确定性（读入'a'时，在状态{1,4}下只有状态 1 能继续转移）。这个小小的 5 状态 NFA 几乎涵盖了 NFA→DFA 转换中的所有核心概念，还顺带演示了"共用接受态会误收 a*b⁺"的陷阱与"子集构造非最小"的现象，非常适合教学。

5. **NFA 模拟和子集构造有什么关系？** — NFA 模拟在**运行时**动态计算状态集（对特定输入串）；子集构造在**编译时**预先计算所有可能的状态集（对所有可能输入）。子集构造本质上是"穷举"了 NFA 模拟可能遇到的所有状态集。因此，如果只需要判断少数几个串，NFA 模拟更高效；如果需要反复判断大量串，构造 DFA 更高效。

6. **如何验证生成的 DFA 是正确的？** — 可以选一组测试串（包括应接受和应拒绝的），同时对 NFA 模拟和 DFA 模拟运行，比较结果。DFA 模拟很简单：从初态 D0 出发，逐字符查表转移，最后检查是否在接受态。如果 NFA 和 DFA 对所有测试串结果一致，则转换正确。
